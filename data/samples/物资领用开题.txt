一、选题背景与意义

高校行政部门与教学单位日常需要申领办公耗材及部分固定资产类物资。若依赖纸质单据或分散电子表格，容易出现库存信息不准确、申领审核周期偏长、出库记录难追溯等问题，既增加仓库管理成本，也不利于资产台账核对与年底盘点。

建设面向校园场景的物资领用信息系统，将物资目录、分类台账、在线申领、审核出库与领用记录集中管理，有助于规范领用流程、提高物资周转效率。本课题拟设计并实现一套基于 B/S 架构的高校办公物资领用管理系统，作为毕业设计工程实践课题，突出需求分析、系统设计与实现能力训练。


二、国内外研究现状简述

企业级 ERP、WMS 以及固定资产管理平台功能较为完整，通常覆盖采购入库、供应商管理、条码或 RFID 盘点、折旧核算乃至财务结算等环节，但部署成本与实施范围往往超出本科毕业设计可承受规模。国内高校信息化建设中，部分单位已提供物资申领或仓库管理入口；毕业设计选题里，基于 Java Web、Spring Boot 实现耗材管理、出入库管理类系统的案例亦较常见，技术路线成熟、可参考性强。

经与指导教师讨论，本课题将研究重点放在「办公物资领用」主路径：物资检索、申领申请、审核出库与基础台账维护。采购流程、第三方支付、全库盘点设备对接等可作为背景对比，不纳入本期必实现范围，以保证在有限周期内完成分析、设计、编码、测试与说明书撰写。


三、研究目标

1. 分析高校办公物资领用业务，梳理申领人与仓库管理侧的使用需求。
2. 完成系统总体设计、模块划分与数据库设计。
3. 采用 Spring Boot 与 Vue 实现可运行系统，覆盖物资目录、分类维护、申领申请与审核出库、领用记录与公告等功能。
4. 完成功能测试，整理源码、数据库脚本与毕业设计说明书，准备答辩。


四、研究内容与拟实现功能

（一）用户与权限

系统面向物资申领人与仓库管理人员。申领人可注册登录、维护个人资料、按分类检索物资并提交领用申请，查看审核进度与公告；管理侧负责物资与分类台账维护、申领审核出库、用户与公告管理。总管与库管等岗位职责在详细设计中划分。

（二）主要功能

1. 用户注册、登录与个人资料维护。
2. 物资分类浏览与条件检索，查看库存余量、规格说明等基本信息。
3. 在线提交物资领用 / 耗材申领申请，填写申领数量与用途说明；查询本人申领记录。
4. 管理侧对待审申领进行通过或驳回；通过后占用对应库存数量（出库）。
5. 支持按业务规则办理退库或取消，并回补库存。
6. 物资目录与分类等资产台账基础信息维护。
7. 用户信息管理。
8. 仓库公告（领用须知、临时停领等）的发布与查阅。
9. 管理端业务概览（待审申领、物资种类、近期出库情况等）。

若进度允许，可补充简易入库登记或申领明细导出；时间不足时，入库侧可采用管理端直接调整库存的方式，不单独展开采购单模块。

（三）研究重点

围绕物资库存数量占用与申领单审核出库 / 回补，以及申领端与管理端的功能边界，形成从选物资、填数量到审核出库与记录查询的基本业务过程。


五、关键问题与解决思路

1. 需求范围控制：开题调研阶段易涉及采购、支付、盘点等扩展能力；本期以申领—审核—出库主流程及必要基础数据为准，其余能力不作为答辩必演示项。
2. 库存一致性：审核通过时扣减库存，取消或退库时回补，防止超领；库存不足时的处理规则在详细设计中约定。
3. 权限与数据安全：区分申领端与管理端，防止越权查看或改动他人申领。
4. 前后端协作：明确接口与页面流程，保证联调测试可执行。


六、技术路线

1. 体系结构：B/S，前后端分离。
2. 后端：Java、Spring Boot、MySQL。
3. 前端：Vue、Element Plus 等。
4. 开发过程：需求分析 → 系统设计 → 编码实现 → 测试修改 → 撰写说明书。


七、进度安排（参考）

第 1–2 周：查阅资料，完成开题，细化需求与用例。
第 3–4 周：完成总体设计、数据库设计与界面草案。
第 5–8 周：实现登录注册、物资目录、申领审核出库、基础数据与公告等模块。
第 9–10 周：系统联调与测试，补充说明材料。
第 11–12 周：完善毕业论文（设计说明书），准备答辩。


八、预期成果

1. 高校办公物资领用管理系统软件一套（含源码与数据库脚本）。
2. 毕业设计说明书一份。
3. 系统演示环境及主要操作说明。
