一、选题背景与意义

在数字化校园建设进程中，教学专题片、公开课录像及各类影视素材呈现快速增长态势，但分散存储、检索困难、权限界定模糊以及观看历史记录管理欠缺等问题日益突出，亟需以规范化的信息管理手段加以解决。为有效整合影视资源目录、规范点播与收藏等交互行为的记录、集中发布校园公告，本课题拟构建一套基于 B/S 架构的校园影视资源点播系统。此举的核心目的在于提高资源利用效率，而非构建复杂的推荐算法。该系统的开发过程将重点锻炼并体现学生从事软件工程实践所应具备的需求分析、系统设计与编码实现能力，因此本课题具有重要的实践教学意义。

二、国内外研究现状简述

在商业流媒体领域，成熟的系统通常包含内容分发网络、多格式转码及会员支付等庞大模块，其部署成本与技术复杂度远远超出了一般本科毕业设计项目所能承担的范围。另一方面，在本科毕业设计选题中，利用 Java Web 技术与 Spring Boot 框架实现同类资源管理系统的案例屡见不鲜，相关技术路线成熟，参考资源丰富。

经与指导教师充分讨论，本课题将研究精力集中于“片库检索、点播请求与交互记录”这一核心业务路径上。至于弹幕实时推送引擎、第三方会员支付对接等具有较高技术门槛或商业属性的扩展能力，将仅作为系统背景或未来展望进行对比阐述，不作为本期开发任务的必备内容，以此确保在有限的周期内，高质量地完成需求分析、架构设计、编码验证、系统测试与毕业设计说明书的撰写工作。

三、研究目标

1.  深入分析校园影视资源点播业务场景，系统梳理普通观众用户与内容管理人员双方的功能诉求。
2.  完成系统的总体架构设计、功能模块划分以及数据库的规范化设计。
3.  依据设计成果，运用 Spring Boot 与 Vue 框架开发一套可运行的校园影视点播系统，核心功能覆盖片库检索、点播请求提交及交互记录查询等主干流程。
4.  对系统进行全面的功能测试，整理并归档完整的项目源码与数据库初始化脚本，形成一篇条理清晰、格式规范的毕业设计说明书，备妥毕业答辩。

四、研究内容与拟实现功能

（一）用户角色与权限管理

本系统面向两类角色设计：普通观众用户与系统内容管理人员。观众用户能够自主完成注册与登录操作，可以完善个人资料，办理核心业务并查阅个人操作历史及公开公告。管理人员则负责系统基础数据的维护、业务流程的审核或确认、用户信息的管理以及系统公告的发布与更新。在后续详细设计阶段，需进一步细化总管理员与各业务岗位的具体职责划分与权限等级。

（二）核心功能模块

1.  用户注册、登录及个人资料（如昵称、联系方式等）的维护。
2.  影视资源的分类浏览与关键词检索。
3.  用户提交点播申请或发起影视资源收藏，并能查询本人的相关历史记录。
4.  管理端对提交的申请单据进行审核、确认或拒绝处理。
5.  影视资源库的基础信息维护与分类目录管理。
6.  用户基本信息的管理与维护。
7.  系统公告的发布与面向所有用户的查阅功能。
8.  管理后台提供基本业务数据概览看板。

（三）研究重点

本课题的研究重点在于理清影视资源档案与用户点播、收藏等业务单据之间的数据流转关系，明确用户端与管理端的功能分割界限，最终形成一条从用户发起业务办理到后台进行状态管理，再到个人历史记录查询的完整且闭环的业务处理过程。

五、关键问题与解决思路

1.  **需求范围控制问题**：在设计调研阶段，容易将扩展性功能（如弹幕引擎、移动端支付等）纳入核心开发计划。**解决思路**：本期开发严格限定范围于“浏览片库→发起点播/收藏请求→管理端审核或确认”的主干流程及其必要的基础数据支撑，其余功能不列入本次答辩必须演示的范畴。
2.  **数据状态一致性问题**：当资源下架后，已发起的关联点播或收藏单据应被及时限制或锁定，避免出现逻辑矛盾。**解决思路**：在数据库设计与后端业务逻辑中设置联动约束与状态校验，确保用户无法对已失效资源办理新的业务。
3.  **权限与数据安全问题**：需严格区分用户端与管理端的功能权限，防止普通用户越权访问管理后台或篡改他人的操作记录。**解决思路**：采用基于角色的访问控制模型，在后端接口层进行统一鉴权，确保数据操作的安全性。
4.  **前后端协作问题**：需保证前端与后端数据交互规范且高效。**解决思路**：在编码前，拟订清晰的 API 接口文档，梳理前端页面流转逻辑，确保后续联调测试能够顺利开展。

六、技术路线

1.  **体系结构**：采用浏览器/服务器模式，实现前后端分离部署。
2.  **后端技术栈**：以 Java 为主要编程语言，选用 Spring Boot 框架搭建后端服务层，以 MySQL 作为关系型数据库进行数据持久化管理。
3.  **前端技术栈**：选用 Vue 框架构建用户界面，并整合 Element Plus 等 UI 组件库以优化开发效率与界面表现。
4.  **开发流程**：严格遵循软件工程的一般步骤，依次完成需求分析、系统设计、编码实现、测试修改及说明书撰写五个迭代环节。

七、进度安排（参考）

第 1–2 周：查阅国内外相关资料，完成开题报告，细化系统需求并绘制用例图。
第 3–4 周：完成系统总体架构设计、数据库概念与逻辑设计，并绘制前端界面原型草图。
第 5–8 周：完成用户身份认证、影视资源管理及业务办理核心模块的编码实现，同步完成公告等基础功能的构建。
第 9–10 周：开展系统内部联调与功能测试，修复缺陷，并根据测试结果补充文档材料。
第 11–12 周：完成毕业论文（毕业设计说明书）的撰写与排版工作，制作答辩演示材料。

八、预期成果

1.  一套完整可运行的校园影视资源点播系统软件（包含项目源码与数据库初始化脚本）。
2.  一份内容详实、符合规范要求的毕业设计说明书。
3.  系统演示环境及关键操作流程的说明文档。
