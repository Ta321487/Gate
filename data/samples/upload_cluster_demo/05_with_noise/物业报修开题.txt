一、选题背景与意义

小区公共设施与室内报修若依赖电话或纸质登记，容易出现响应慢、进度难查、投诉难闭环等问题。

建设物业报修信息系统，将报修类型、在线报修与受理、记录与公告集中管理，有助于提升物业服务体验。本课题拟设计并实现一套基于 B/S 架构的小区物业报修管理系统，作为毕业设计工程实践课题，突出需求分析、系统设计与实现能力训练。


二、国内外研究现状简述

商业物业 SaaS 常含收费、停车、访客与工单一体化，但部署成本与业务范围往往超出本科毕业设计可承受规模。国内毕业设计选题里，基于 Java Web、Spring Boot 实现同类系统的案例较为常见，技术路线成熟、可参考性强。

经与指导教师讨论，本课题将研究重点放在「报修申请—受理—完成」主路径。物业费在线支付与访客门禁硬件等可作为背景对比，不纳入本期必实现范围，以保证在有限周期内完成分析、设计、编码、测试与说明书撰写。


三、研究目标

1. 分析小区物业报修业务，梳理业主 / 住户与管理侧的使用需求。
2. 完成系统总体设计、模块划分与数据库设计。
3. 采用 Spring Boot 与 Vue 实现可运行系统，覆盖报修申请—受理—完成等功能。
4. 完成功能测试，整理源码、数据库脚本与毕业设计说明书，准备答辩。


四、研究内容与拟实现功能

（一）用户与权限

系统面向业主 / 住户与物业管理人员。业主 / 住户可注册登录、维护个人资料、办理主业务并查看本人记录与公告；管理侧负责基础数据维护、业务审核或办理、用户与公告管理。总管与业务岗位职责在详细设计中划分。

（二）主要功能

1. 用户注册、登录与个人资料维护。
2. 选择报修类型提交报修并填写说明。
3. 查询本人报修记录。
4. 管理侧受理、处理与完结。
5. 报修类型与站点信息维护。
6. 用户信息管理。
7. 公告发布与查阅。
8. 管理端业务概览。
9. 若进度允许，可补充强制附件。

（三）研究重点

围绕报修单状态流转与类型基础数据，以及用户端与管理端的功能边界，形成从办理到记录查询的基本业务过程。


五、关键问题与解决思路

1. 需求范围控制：开题调研阶段易涉及物业费在线支付与访客门禁硬件等扩展能力；本期以提交报修 → 受理 → 完成主流程及必要基础数据为准，其余能力不作为答辩必演示项。
2. 工单状态一致性：受理与完结可追溯。
3. 权限与数据安全：区分用户端与管理端，防止越权查看或改动他人数据。
4. 前后端协作：明确接口与页面流程，保证联调测试可执行。


六、技术路线

1. 体系结构：B/S，前后端分离。
2. 后端：Java、Spring Boot、MySQL。
3. 前端：Vue、Element Plus 等。
4. 开发过程：需求分析 → 系统设计 → 编码实现 → 测试修改 → 撰写说明书。


七、进度安排（参考）

第 1–2 周：查阅资料，完成开题，细化需求与用例。
第 3–4 周：完成总体设计、数据库设计与界面草案。
第 5–8 周：实现登录注册、主业务模块、基础数据与公告等。
第 9–10 周：系统联调与测试，补充说明材料。
第 11–12 周：完善毕业论文（设计说明书），准备答辩。


八、预期成果

1. 小区物业报修管理系统软件一套（含源码与数据库脚本）。
2. 毕业设计说明书一份。
3. 系统演示环境及主要操作说明。
