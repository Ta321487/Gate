本科毕业设计（论文）开题报告

题目：基于 Spring Boot 的养老机构健康监测与照护管理系统的设计与实现

一、选题背景与意义

1.1 选题背景
我国人口老龄化程度持续加深，机构养老与医养结合需求不断增长。养老机构需要同时承担老年人日常生活照护、健康监测、慢性病管理、感染防控、紧急事件处置与家属沟通等工作。传统依赖纸质护理记录和经验式管理的方式，容易出现健康异常发现滞后、重点老人关注不足、交接班信息不完整、物资耗材管理混乱、管理人员难以全局掌握机构运行状态等问题。

在“智慧养老”“医养结合”政策推动下，信息化成为提升养老服务质量的重要路径。针对中小型养老机构建设一套成本可控、功能聚焦的健康监测与照护管理系统，具有较强应用价值。

1.2 选题意义
（1）服务老龄化社会真实需求，选题具有人文关怀与时代性；
（2）将健康打卡、重点老人管理、异常流转、物资保障与统计决策结合；
（3）基于 Spring Boot 完成完整 Web 系统工程训练；
（4）贴近当前毕业设计热点方向。

二、国内外研究现状

发达国家智慧养老起步较早，常见远程监护、可穿戴设备与照护记录电子化。国内养老信息化发展迅速，但两极分化明显：大型机构系统完善，中小机构仍大量手工管理。本课题不强调硬件采集，聚焦业务管理闭环与数据可视化，构建轻量可用的机构端系统。

三、研究目标与主要内容

3.1 研究目标
基于 Spring Boot 设计实现养老机构健康监测与照护管理系统，支持管理员、护士/照护员、医生（可选）等角色，实现老人建档、每日健康打卡、重点关注老人管理、异常观察对象管理、转归记录、检查评估登记、防护与生活物资管理、综合看板。

3.2 主要内容
需求分析、总体设计、数据库设计、功能开发、权限安全、系统测试、论文撰写。

3.3 关键问题
（1）老年人多状态管理（正常在住、重点关注、隔离观察、住院、退住等）；
（2）照护员高频打卡与异常自动汇总；
（3）敏感信息与最小权限访问；
（4）机构运营指标可视化。

四、技术方案与系统架构

4.1 技术选型
Spring Boot、Spring Security、MyBatis、MySQL、Thymeleaf、AdminLTE、ECharts、PageHelper、Druid、Maven；JDK 8/11。

4.2 系统架构
B/S 分层架构：Controller、Service、Dao、Entity；前端按业务模块组织页面；数据库以老人主索引关联各类记录。

4.3 非功能目标
界面简洁、核心操作路径短；支持分页查询；关键操作有权限控制；演示环境稳定可运行。

五、系统功能模块设计

5.1 机构运行看板
在住老人数、今日已打卡/未打卡、体温或血压异常数、重点关注人数、本周就医数、物资低库存预警。

5.2 用户权限管理
管理员分配照护员账号；不同角色看到不同菜单。

5.3 每日健康打卡
体温、血压、血糖、饮食睡眠、精神状态、跌倒风险等；支持按楼层/房间筛选未打卡老人。

5.4 重点关注老人管理
失能失智、高风险慢性病、近期术后、传染病观察期等老人专项管理。

5.5 异常观察对象管理
新入住观察、疑似感染观察、情绪波动需关注对象等。

5.6 转归管理
康复稳定、转院就医、退住、死亡登记等结局管理。死亡登记仅作为必要台账能力，界面需严肃规范，演示数据脱敏。

5.7 检查检验与评估记录
体检结果、简化评估量表、相关检测登记。

5.8 物资与耗材管理
口罩、消毒液、护理耗材、生活物资出入库。

5.9 专题分析页
体检完成情况、异常趋势、耗材消耗统计。

六、数据库设计思路

老人基本信息、健康打卡、重点关注、观察对象、转归、检查评估、物资、用户角色、楼栋房间字典等。以老人 ID 为主关联键，状态字段驱动列表筛选。

七、可行性分析

Spring Boot 生态成熟；经济成本低；界面符合机构后台管理习惯；采用模拟老人数据，规避真实隐私风险。

八、实施计划

第1–2周：文献阅读与开题；
第3–4周：照护流程调研，输出需求与 ER 图；
第5–9周：用户权限、老人档案、打卡、重点/观察对象；
第10–12周：转归、检查记录、物资、看板；
第13–14周：测试与演示数据；
第15–16周：论文定稿与答辩。

九、预期成果

可运行系统、设计文档、数据库脚本、毕业论文、测试报告与答辩 PPT。

十、创新点与特色

（1）面向老龄化场景的健康照护信息化；
（2）日常监测与异常闭环并重；
（3）照护角色权限贴合机构组织方式；
（4）可视化支撑机构运营决策。

十一、可能问题与对策

医疗专业性强：字段保持必要且克制，不做成完整电子病历；
敏感功能：权限限制与二次确认；
物联网设备接入：作为论文展望，不作为必做项。
