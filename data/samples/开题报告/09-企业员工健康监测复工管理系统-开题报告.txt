本科毕业设计（论文）开题报告

题目：基于 Spring Boot 的企业员工健康监测与复工管理系统的设计与实现

一、选题背景与意义

1.1 选题背景
在制造业、园区办公、连锁服务业等组织中，员工健康状况直接影响出勤稳定性、生产连续性与合规管理水平。企业在流感季、突发公共卫生事件、高温作业季等场景下，需要组织健康打卡、识别异常员工、追踪同班次暴露人员、评估复工条件，并保障口罩消毒液等防护物资供应。

依赖微信群与表格汇总的方式，难以满足高频采集、分级权限、历史追溯与管理层可视化决策。建设企业员工健康监测与复工管理系统，可把“监测—分类管控—复工评估—物资保障—统计分析”串成闭环。

1.2 选题意义
（1）面向企业管理信息化，应用面广；
（2）健康监测与复工评估结合，业务完整；
（3）基于 Spring Boot 技术栈完成规范化系统开发；
（4）便于结合企业实践背景进行答辩说明。

二、国内外研究现状

国外常将员工健康纳入 EHS（环境健康安全）或人力资源信息系统；国内健康打卡与企业应急管理工具逐步普及，目前更强调常态化能力沉淀与数据安全。中小企业需要轻量、易部署的 Web 管理系统，本课题以此为切入点。

三、研究目标与主要内容

3.1 研究目标
基于 Spring Boot 实现员工档案与部门管理、健康打卡、异常员工管理、暴露风险人员管理、复工评估登记、检测记录、物资领用、部门统计看板与 RBAC 权限控制。

3.2 主要内容
需求分析、系统设计、数据库设计、功能开发、测试、论文撰写。

3.3 关键问题
部门数据范围控制；员工状态流转；复工条件核验；管理驾驶舱指标设计。

四、技术方案与系统架构

4.1 技术选型
Spring Boot、Spring Security、MyBatis、MySQL、Thymeleaf、AdminLTE、ECharts、PageHelper、Druid、Maven。

4.2 架构设计
B/S 分层架构，按组织权限、健康监测、风险管控、复工管理、物资保障、统计分析划分模块。

4.3 安全方案
登录认证、角色授权、越权拦截；演示环境使用模拟员工数据，注意隐私表述。

五、系统功能模块设计

5.1 健康复工驾驶舱
打卡率、异常人数、暴露观察人数、待复工评估、物资预警、部门对比图。

5.2 用户权限管理
系统管理员、部门管理员、普通员工。

5.3 健康打卡
体温、症状、相关风险信息、是否适合到岗；支持记录查询。

5.4 异常员工管理
不适/就诊/住院等在管员工建档与措施记录。

5.5 暴露风险人员管理
同部门、同班次、同车间观察名单，可转异常或解除。

5.6 复工与终止在管
复工登记为主；可支持长病假/离职导致的在管终止。

5.7 检测登记
抗原/核酸/体检等相关结果登记。

5.8 物资领用
防护物资出入库与个人领用记录。

5.9 专题页
检测专题、物资专题等统计分析。

六、数据库设计要点

员工、部门、打卡、异常员工、暴露人员、复工单、检测、物资、用户角色。员工工号建议唯一。

七、可行性分析

Spring Boot 技术成熟；企业场景清晰；使用模拟数据；注意劳动隐私合规表述。

八、进度安排

开题2周，设计2周，开发7周，测试2周，论文答辩3周。

九、预期成果

可运行系统、毕业论文、数据库脚本、测试记录、答辩材料。

十、创新点与特色

（1）企业健康监测与复工评估闭环；
（2）部门维度统计分析；
（3）打卡与风险人员联动；
（4）Spring Security 支撑分级权限。

十一、风险与对策

制度差异大：做成可配置状态；
移动端：Web 优先；
复杂审批：简化为部门管理员确认。
