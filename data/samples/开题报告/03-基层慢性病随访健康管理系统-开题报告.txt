本科毕业设计（论文）开题报告

题目：基于 Spring Boot 的基层慢性病患者随访与健康管理系统的设计与实现

一、选题背景与意义

1.1 选题背景
高血压、糖尿病、冠心病等慢性病具有患病率高、病程长、需长期管理的特点。基层医疗卫生机构承担建档立卡、定期随访、用药指导、并发症筛查与健康教育等任务。现实中仍大量依赖纸质随访表与分散表格，导致随访遗漏、指标趋势难看、异常患者难预警、管理人员统计负担重。

在“健康中国”与分级诊疗背景下，基层慢性病精细化管理需求持续上升。利用信息化系统实现患者全周期管理、随访打卡、高风险患者标记、转归记录与数据可视化，有助于提升慢病管理质量与工作效率。

1.2 选题意义
（1）服务基层医疗与公卫服务实际场景，选题具有时代性；
（2）将长期随访流程信息化，形成可量化管理；
（3）基于 Spring Boot 技术栈完成完整系统分析、设计与实现；
（4）贴近当前毕设选题热点，社会意义明确。

二、国内外研究现状

国外慢病管理强调患者自我管理、远程监测与连续性医疗记录；国内家庭医生签约、基本公卫服务项目推动了慢病管理信息化，但基层机构系统能力参差不齐。仍有大量中小场景需要轻量级、易部署、功能聚焦的管理系统。本课题定位于可落地的基层随访管理平台。

三、研究目标与主要内容

3.1 研究目标
构建基于 Spring Boot 的基层慢性病随访与健康管理系统，实现患者建档、分类管理、随访打卡、异常指标记录、高风险随访对象管理、转归管理、健康物资发放与统计看板。

3.2 主要内容
需求建模、系统设计、数据库设计、模块开发、安全权限、测试验证、论文撰写。

3.3 关键问题
（1）随访计划与打卡记录如何关联患者档案；
（2）如何标识高风险患者并支持重点跟进；
（3）如何展示血压、血糖等指标变化趋势；
（4）如何保证医生、公卫人员、管理员等角色权限分离。

四、技术方案与系统架构

4.1 技术选型
Spring Boot、Spring Security、MyBatis、MySQL、Thymeleaf、AdminLTE、jQuery、ECharts、PageHelper、Druid、Maven；JDK 8/11。

4.2 架构设计
B/S 三层结构。Controller 处理请求，Service 处理随访业务规则，Dao/Mapper 访问数据库，前端负责表单、列表与图表展示。

4.3 安全设计
基于 Spring Security 的认证授权与菜单控制；演示数据虚构，论文中声明隐私保护措施。

五、系统功能模块设计

5.1 慢病管理驾驶舱
在管患者数、本月应访/已访、异常打卡数、重点患者数、控制达标率、物资预警。

5.2 用户权限管理
系统账号维护与角色分配。

5.3 随访与健康打卡
录入血压、血糖、症状、用药依从性等；支持按姓名/日期查询；异常值高亮。

5.4 重点患者管理
合并症多、控制不佳、近期住院等高风险患者专项管理。

5.5 待观察/新纳入对象管理
新筛查对象、待复核建档对象等名单管理。

5.6 转归管理
病情稳定退出重点管理、转诊上级医院、死亡登记等。

5.7 检查检验记录
登记血糖化验、糖化血红蛋白、血脂等检查结果。

5.8 健康物资管理
血压计借还、宣教手册、慢病管理包发放记录。

5.9 专题分析
随访完成率、异常趋势、物资发放统计。

六、数据库设计要点

患者基本信息、慢病档案、随访计划、打卡记录、重点患者、观察对象、检查记录、转归记录、物资发放、用户角色等。以患者 ID 关联全周期事件。

七、可行性分析

技术可行、成本低、界面友好；使用模拟病例数据，符合毕设演示要求；不替代诊疗决策，只做管理辅助。

八、研究方法与进度安排

文献研究、调研分析、迭代开发、测试验证。

第1–2周：文献与开题；
第3–4周：需求、流程、ER 图；
第5–8周：框架搭建、权限、档案、打卡；
第9–11周：重点患者、检查、转归、物资、看板；
第12–13周：测试优化；
第14–16周：论文与答辩。

九、预期成果

系统源码、数据库脚本、毕业论文、测试用例、答辩材料。

十、创新点与特色

（1）面向基层慢病长期管理设计；
（2）风险分层与随访闭环结合；
（3）打卡数据驱动预警与统计；
（4）Spring Boot + Security 保障系统安全与可维护性。

十一、可能存在的问题与对策

医学指标复杂：优先实现血压、血糖等核心字段；
随访规则多变：采用简化状态管理；
消息提醒复杂：以“今日待随访列表”替代推送。
