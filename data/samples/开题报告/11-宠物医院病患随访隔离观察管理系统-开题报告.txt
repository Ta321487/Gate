本科毕业设计（论文）开题报告

题目：基于 Spring Boot 的宠物医院病患随访与隔离观察管理系统的设计与实现

一、选题背景与意义

1.1 选题背景
宠物经济快速发展，宠物医院在日常诊疗之外，还需要处理传染病隔离观察、术后随访、疫苗接种提醒、住院监护与耗材管理等工作。许多中小型宠物医院仍使用纸质病历与微信沟通，导致复诊跟进不及时、隔离动物状态不清、接触动物排查困难、库存耗材混乱、管理者难以掌握门店医疗风险数据。

将信息化引入宠物医疗随访与隔离管理，有助于提升服务质量、降低漏访风险，并形成可追溯的医疗管理台账。该选题面向新兴行业场景，同时适合作为 Spring Boot Web 系统毕业设计方向。

1.2 选题意义
（1）面向宠物医疗场景，选题新颖，答辩辨识度高；
（2）业务流程完整，能支撑本科工作量；
（3）Spring Boot 技术栈经典，工程可落地；
（4）可体现领域建模与管理系统设计能力。

二、国内外研究现状

国外宠物医疗信息化（PIMS）较成熟，常见预约、电子病历、库存与提醒一体化。国内宠物医疗连锁加速数字化，但大量单体门店系统能力仍弱。本课题借鉴电子病历与随访管理方法，结合 Web 技术实现轻量门店版系统。

三、研究目标与主要内容

3.1 研究目标
基于 Spring Boot 设计实现宠物医院病患随访与隔离观察管理系统，支持前台/医师/管理员角色，完成宠物建档、每日观察打卡、传染病隔离病患管理、同笼/同院接触动物排查、康复转归、检查检验登记、疫苗与耗材物资、数据看板。

3.2 主要内容
需求分析、架构与数据库设计、模块开发、权限控制、测试、论文。

3.3 关键问题
宠物—主人关联建模；隔离状态流转；随访提醒（可用待办列表替代消息推送）；库存与医疗主数据协同。

四、技术方案与系统架构

4.1 技术选型
Spring Boot、Spring Security、MyBatis、MySQL、Thymeleaf、AdminLTE、ECharts、Maven。

4.2 架构设计
B/S 分层架构，模块包括档案管理、观察打卡、隔离管理、检测转归、物资库存、统计分析、系统权限。

五、系统功能模块设计

5.1 门店运营与医疗看板
在院宠物数、隔离中数量、今日未打卡观察、待复诊、耗材预警。

5.2 用户权限管理
院长/管理员、医师、助理。

5.3 观察打卡
隔离区或住院宠物体温、精神、饮食排便记录。

5.4 重点病患管理
传染病/重症/术后重点病例管理。

5.5 接触动物管理
同病房同笼具接触者观察。

5.6 转归管理
治愈出院、病危转诊、死亡登记（严肃权限控制）。

5.7 检测登记
血检、病原检测结果简项登记（非完整检验流程）。

5.8 物资与疫苗管理
疫苗、药品、耗材库存。

5.9 专题分析
疫苗接种、检测、物资专题统计。

六、数据库设计要点

宠物档案、主人信息、就诊/住院单、观察打卡、隔离病患、接触观察、检测登记、转归、物资、用户角色。

七、可行性分析

技术可行、趣味性强、模拟数据易准备；专业表述适度，不过度兽医化。

八、进度安排

开题2周 → 设计2周 → 开发7周 → 测试2周 → 论文答辩3周。

九、预期成果

可运行系统、库脚本、论文、测试与演示材料。

十、创新点与特色

（1）宠物医疗随访与隔离观察信息化；
（2）隔离观察与物资管理一体化；
（3）可视化兼顾经营与医疗视角；
（4）Spring Boot + Security 保障系统安全。

十一、风险与对策

专业术语多：对齐常用管理字段；
死亡敏感功能：权限与二次确认；
预约排班、完整检验检查流程：不在本期，可作为后续扩展。
