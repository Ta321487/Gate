本科毕业设计（论文）开题报告

题目：基于 Spring Boot 的社区公共卫生事件应急管理系统的设计与实现

一、选题背景与意义

1.1 选题背景
随着城镇化加速与人口流动加剧，社区作为社会治理的基本单元，在突发公共卫生事件应对中承担着信息上报、人员排查、健康监测、物资保障与风险闭环管理等关键职责。传统依靠纸质台账、微信群通知与 Excel 汇总的方式，存在信息滞后、口径不一、责任难追溯、统计困难等问题，难以满足基层“早发现、早报告、早隔离、早处置”的工作要求。

近年来，国家持续推进基层治理数字化与公共卫生应急能力建设，强调将信息化手段嵌入社区日常管理与应急响应流程。面向社区场景构建一套轻量、可落地、权限清晰、流程闭环的公共卫生事件应急管理系统，具有明确的现实需求与应用价值。

1.2 选题意义
（1）理论意义：将应急管理流程与信息系统工程方法相结合，探索“日常监测—风险识别—人员管控—物资保障—数据研判”一体化的信息化模型。
（2）实践意义：为社区工作人员提供统一业务入口，实现重点人员管理、健康打卡、接触者排查、转归跟踪、物资台账与数据看板，提升响应效率与数据准确性。
（3）工程训练意义：综合运用 Spring Boot、Spring Security、MyBatis、MySQL 等技术完成完整系统开发，覆盖需求分析、数据库设计、权限控制、业务实现与测试部署，符合计算机类本科毕业设计目标。

二、国内外研究现状

2.1 国外研究现状
发达国家在公共卫生信息化方面起步较早，常见做法是将病例监测、接触者追踪、实验室检测、疫苗接种与物资调配纳入统一信息平台，并强调数据标准、隐私保护与跨部门协同。部分方案结合移动端打卡与可视化仪表盘提升决策效率。

2.2 国内研究现状
国内基层公卫与社区管理系统建设持续推进，在人员管理、健康监测与应急物资保障等方面积累了实践经验。现有系统在基层场景适配、权限分级、流程闭环、可视化研判等方面仍有提升空间，尤其需要面向街道、社区可落地的轻量级方案。

2.3 本课题切入点
本课题面向社区公卫应急场景，构建结构清晰、模块可扩展、角色权限明确、具备基础可视化能力的管理系统，强调日常管理与应急响应的业务连续性。

三、研究目标与主要内容

3.1 研究目标
设计并实现一套基于 Spring Boot 的社区公共卫生事件应急管理系统，实现重点人员全生命周期管理、健康打卡与核查、风险人员流转、物资库存管理、统计看板及基于角色的访问控制，形成可演示、可测试的完整工程成果。

3.2 主要内容
（1）需求分析：梳理社区工作人员、网格员、管理员等角色需求；
（2）总体设计：完成系统架构、模块划分、数据库设计与安全方案；
（3）功能实现：完成用户权限、健康打卡、重点人员、接触者排查、检测记录、物资管理、首页统计与可视化看板；
（4）系统测试：功能测试、权限测试与基本性能验证；
（5）论文撰写：完成设计说明、关键技术、实现与总结。

3.3 拟解决的关键问题
（1）人员状态流转（观察中→确诊/重点管控→治愈/解除等）的可追溯建模；
（2）基于角色的敏感健康数据访问控制；
（3）打卡、排查、物资与统计数据的关联查询与可视化；
（4）系统模块划分清晰，便于后续扩展。

四、技术方案与系统架构

4.1 技术选型
（1）后端框架：Spring Boot
（2）安全框架：Spring Security
（3）持久层：MyBatis + PageHelper
（4）数据库：MySQL
（5）连接池：Druid
（6）前端：Thymeleaf + Bootstrap/AdminLTE + jQuery + ECharts
（7）构建工具：Maven
（8）开发环境：JDK 8/11、IntelliJ IDEA

4.2 系统架构
采用 B/S 架构，分为表现层、业务层、持久层与数据层：
表现层负责页面渲染、表单交互与图表展示；
业务层封装人员管理、打卡核查、状态流转、物资出入库与统计分析；
持久层完成实体映射与 SQL 访问；
数据层由 MySQL 存储业务与权限数据。

4.3 安全方案
基于 Spring Security 实现登录认证与授权；按管理员、业务员、普通用户分配菜单与接口权限；对非法访问进行拦截跳转；关键操作保留基础审计信息。

五、系统功能模块设计

5.1 系统首页与数据看板
展示打卡人数、重点人员数量、新增风险人员、物资预警等指标，提供趋势图与分类统计。

5.2 用户与权限管理
用户增删改查、角色分配、密码修改、登录注销。

5.3 健康打卡管理
个人每日健康打卡（体温、症状、出行情况等）；打卡记录查询与异常筛查。

5.4 重点人员管理
人员基础信息建档；观察对象与重点管控对象分类管理；详情查看与状态更新。

5.5 风险接触者管理
登记接触史、观察期限；支持转为重点人员或解除观察；按条件检索。

5.6 转归管理
治愈/康复、不良结局等登记，保证人员状态闭环可查。

5.7 检测与核查记录
登记相关检测结果并与人员档案关联，支持历史查询。

5.8 应急物资管理
防护与保障类物资入库出库、库存查询、低库存提醒。

5.9 可视化专题页
检测覆盖、物资保障等专题展示，增强决策支持。

六、数据库设计思路

核心实体包括：用户与权限、人员基础信息、重点人员、风险接触者、打卡记录、检测记录、转归记录、物资与出入库记录等。
设计原则：主键明确、关联清晰、状态字段可枚举、关键操作可追溯、列表查询支持分页。

七、可行性分析

7.1 技术可行性：Spring Boot 生态成熟，资料丰富，可在毕设周期内完成。
7.2 经济可行性：采用开源技术与实验环境部署，成本可控。
7.3 操作可行性：后台管理界面清晰，经简单培训即可使用。
7.4 伦理可行性：采用模拟数据，强调隐私保护与最小权限原则。

八、研究方法与实施计划

8.1 研究方法
文献调研、需求调研、原型开发、模块化实现、黑盒测试。

8.2 进度安排（可按校历调整）
第1–2周：选题调研、查阅文献、撰写开题报告；
第3–4周：需求分析、流程图、数据库设计；
第5–8周：搭建 Spring Boot 框架，完成用户权限与基础业务；
第9–11周：完成打卡、人员流转、物资与统计看板；
第12–13周：联调测试与缺陷修复；
第14–15周：撰写论文初稿与系统说明；
第16周：论文修改、答辩准备。

九、预期成果

（1）可运行的社区公共卫生事件应急管理系统；
（2）数据库脚本、源代码与部署说明；
（3）毕业设计论文；
（4）答辩演示材料与测试用例。

十、创新点与特色

（1）面向社区场景的公卫应急管理闭环设计；
（2）以人员状态流转为核心组织业务；
（3）打卡监测、风险排查、物资保障与可视化一体化；
（4）基于 Spring Security 的角色权限控制。

十一、可能存在的问题与对策

（1）需求边界过大：优先实现核心闭环，扩展功能写入展望；
（2）可视化经验不足：采用 ECharts 等成熟组件；
（3）安全细节复杂：先保证认证授权可用，再逐步完善。
