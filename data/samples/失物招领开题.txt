一、选题背景与意义

校园内遗失校园卡、证件、数码配件与生活用品的情况较为常见。传统失物招领依赖公告栏、保卫处登记本或社交群转发，信息分散、检索困难、认领对接效率低，也容易造成二次遗失或冒领风险。建设统一的失物招领信息系统，将启事发布、在线认领申请与审核领取集中管理，有助于提高寻物效率、规范招领流程。

本课题拟设计并实现一套面向校园场景的失物招领系统，覆盖启事目录、认领申请与审核及基础信息管理，作为毕业设计工程实践课题。


二、国内外研究现状简述

失物招领与物品公示类系统在部分高校一站式服务中已有入口；毕业设计选题里，基于 Java Web、Spring Boot 实现失物招领、寻物启事管理的案例较为常见。本课题将研究范围限定在 Web 端的启事浏览、认领申请与审核闭环，不扩展图像识别自动匹配、社交即时通讯等复杂能力，保证可按时交付。


三、研究目标

1. 分析校园失物招领业务，梳理普通用户与招领管理人员的使用需求。
2. 完成系统总体设计、模块划分与数据库设计。
3. 采用 Spring Boot 与 Vue 实现可运行系统，覆盖失物检索、认领申请、审核领取、分类与公告等功能。
4. 完成功能测试，整理源码、数据库脚本与毕业设计说明书。


四、研究内容与拟实现功能

（一）用户与权限

系统面向师生用户与招领管理人员。用户可注册登录、维护个人资料、浏览失物 / 招领启事并提交认领申请，查看审核进度与公告；管理侧负责启事与分类维护、认领审核、用户与公告管理。

（二）主要功能

1. 用户注册、登录与个人资料维护。
2. 失物启事分类浏览与条件检索，查看物品特征与拾获地点等信息。
3. 在线提交认领申请，查询本人认领记录。
4. 管理侧对待审认领进行通过或驳回，并记录审核说明。
5. 启事与分类等基础信息维护（含是否可认领状态）。
6. 用户信息管理。
7. 招领公告的发布与查阅。
8. 管理端业务概览（待审认领、启事数量等）。

（三）研究重点

围绕启事公示与认领单状态变化（申请、审核、已认领 / 驳回），以及用户端与管理端功能边界，形成从浏览启事到认领审核的基本业务过程。


五、关键问题与解决思路

1. 需求范围控制：以启事目录、认领申请—审核主流程为主，不扩展自动图像比对、即时通讯等能力。
2. 认领互斥：同一启事在审核通过后应避免重复认领，设计中约定库存 / 可认领标记规则。
3. 权限与隐私：认领审核需核对特征说明，避免越权查看他人敏感联系方式的不当展示。
4. 前后端协作：明确接口与页面流程，保证联调测试可执行。


六、技术路线

1. 体系结构：B/S，前后端分离。
2. 后端：Java、Spring Boot、MySQL。
3. 前端：Vue、Element Plus 等。
4. 开发过程：需求分析 → 系统设计 → 编码实现 → 测试修改 → 撰写说明书。
