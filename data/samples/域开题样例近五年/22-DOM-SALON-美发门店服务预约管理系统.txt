本科毕业设计（论文）开题报告

题目：基于 Spring Boot 与 Vue 的美发门店服务预约管理系统的设计与实现

开题年份：2023

一、选题背景与意义

1.1 选题背景
本人家乡经营一家社区美发店，高峰时段顾客到店排队，理发师空档却出现在非高峰。电话预约记录在本子上，偶尔记错时间引发投诉。店主希望顾客在线查看服务项目与理发师可约时段。

美发、烫染等项目耗时不同，需要按服务项目匹配时段。系统应支持服务档案、理发预约、选时段与到店办结。

1.2 选题意义
（1）平衡理发师负载，减少顾客等待；
（2）预约记录数字化，降低错约；
（3）门店可统计热门服务项目；
（4）训练本地生活服务预约系统开发。

二、国内外研究现状

美团丽人等聚合平台抽佣高，小店可用自建预约页。本科毕设实现 Web 版服务预约即可，不做真支付。

三、研究目标与主要内容

3.1 研究目标
实现美发/美容服务项目浏览、理发预约选时段、到店办结及公告管理。

3.2 主要功能
1. 顾客与店员登录；
2. 美发、美容服务项目与价格档案；
3. 理发师可预约时段展示；
4. 顾客选择服务与时段提交预约；
5. 取消或改约；
6. 到店服务后办结；
7. 公告与促销通知。

3.3 拟解决关键问题
（1）同一理发师同一时段不重复预约；
（2）服务时长与时段长度匹配（简化规则）；
（3）预约提醒通过系统消息或列表展示。

四、技术方案

后端采用 Spring Boot 构建 REST 接口，Spring Security 实现登录鉴权与角色权限；持久层使用 MyBatis 访问 MySQL 数据库。前端采用 Vue 3 与 Element Plus 搭建管理端与业务页面，ECharts 用于工作台等业务统计图表。系统按表现层、业务层、数据访问层分层设计，便于模块开发与联调。

五、非本期范围

真支付、会员储值硬件、短信营销平台不在本期。

六、进度安排

第 1–2 周：文献调研、需求梳理并完成开题答辩；
第 3–4 周：完成系统总体设计、数据库 ER 图与接口说明；
第 5–10 周：分模块编码实现核心业务与权限控制；
第 11–12 周：功能测试、缺陷修复与演示数据准备；
第 13–16 周：撰写毕业论文、整理答辩 PPT 与操作说明。

七、预期成果

（1）可本地部署运行的 Web 管理系统；
（2）MySQL 建表脚本与示例数据；
（3）符合学院规范的本科毕业论文一份；
（4）系统演示视频或答辩材料。
