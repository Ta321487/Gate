本科毕业设计（论文）开题报告

题目：基于 Spring Boot 与 Vue 的校园二手商品交易管理系统的设计与实现

开题年份：2024

一、选题背景与意义

1.1 选题背景
毕业季校园闲置书籍、电子数码、生活用品转让需求大，学生目前主要在校内论坛或微信群交易，商品信息分散，买卖双方缺少订单记录，纠纷时难以举证。

希望有校园二手交易平台：卖家发布商品，买家加入购物车、下单生成订单（演示环境不做真实支付），并可收藏感兴趣的商品、留言咨询。

1.2 选题意义
（1）方便校内闲置物品流转；
（2）订单记录规范交易过程；
（3）收藏与留言提升匹配效率；
（4）训练电商交易流程 Web 实现。

二、国内外研究现状

闲鱼等综合二手平台非校园封闭场景。本科毕设常见购物车下单演示，不对接微信/支付宝真实支付。

三、研究目标与主要内容

3.1 研究目标
实现二手商品浏览、购物车、下单订单管理、收藏与访客留言及公告。

3.2 主要功能
1. 用户登录与个人资料；
2. 二手商品分类发布与详情浏览；
3. 购物车增删与提交订单；
4. 订单状态管理（待确认/已完成等，演示无真支付）；
5. 商品收藏与取消收藏；
6. 访客留言咨询；
7. 公告发布。

3.3 拟解决关键问题
（1）下单后商品库存或状态更新；
（2）购物车与订单数据一致；
（3）演示环境明确标注线下自行交割。

四、技术方案

后端采用 Spring Boot 构建 REST 接口，Spring Security 实现登录鉴权与角色权限；持久层使用 MyBatis 访问 MySQL 数据库。前端采用 Vue 3 与 Element Plus 搭建管理端与业务页面，ECharts 用于工作台等业务统计图表。系统按表现层、业务层、数据访问层分层设计，便于模块开发与联调。

五、非本期范围

微信支付、支付宝真实对接、物流快递 API、信用评价体系不在本期。

六、进度安排

第 1–2 周：文献调研、需求梳理并完成开题答辩；
第 3–4 周：完成系统总体设计、数据库 ER 图与接口说明；
第 5–10 周：分模块编码实现核心业务与权限控制；
第 11–12 周：功能测试、缺陷修复与演示数据准备；
第 13–16 周：撰写毕业论文、整理答辩 PPT 与操作说明。

七、预期成果

（1）可本地部署运行的 Web 管理系统；
（2）MySQL 建表脚本与示例数据；
（3）符合学院规范的本科毕业论文一份；
（4）系统演示视频或答辩材料。
