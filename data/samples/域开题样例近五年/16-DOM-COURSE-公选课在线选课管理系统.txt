本科毕业设计（论文）开题报告

题目：基于 Spring Boot 与 Vue 的公选课在线选课管理系统的设计与实现

开题年份：2023

一、选题背景与意义

1.1 选题背景
我校公选课采用先到先得，以往教务老师开放 Excel 报名或简易表单，高人气课程秒满后仍有人反复提交，且无法校验与学生已有课表是否时间冲突。

选课结果需教务审核后占用名额并写入名单。学生希望清晰看到课程学分、上课时间、剩余名额与选课状态。

1.2 选题意义
（1）公平占用公选课名额；
（2）课表时段冲突自动提示；
（3）选课申请与审核流程透明；
（4）体现教务选课场景建模能力。

二、国内外研究现状

完整教务系统含排课、考务，本科毕设常只做公选课报名与名额管理轻量模块。

三、研究目标与主要内容

3.1 研究目标
实现公选课档案浏览、选课报名、名额占用与审核、时段冲突检测及公告管理。

3.2 主要功能
1. 学生登录与学籍资料；
2. 公选课/课程档案发布（学分、时间、容量）；
3. 学生提交选课报名申请；
4. 教务审核并占用名额；
5. 与已选课程时段冲突检测提示；
6. 选课结果查询；
7. 公告发布。

3.3 拟解决关键问题
（1）名额扣减与审核在同一逻辑中完成；
（2）冲突课程不可同时审核通过；
（3）课程满员后关闭报名入口。

四、技术方案

后端采用 Spring Boot 构建 REST 接口，Spring Security 实现登录鉴权与角色权限；持久层使用 MyBatis 访问 MySQL 数据库。前端采用 Vue 3 与 Element Plus 搭建管理端与业务页面，ECharts 用于工作台等业务统计图表。系统按表现层、业务层、数据访问层分层设计，便于模块开发与联调。

五、非本期范围

智能排课、自动排考、与正方教务深度对接不在本期。

六、进度安排

第 1–2 周：文献调研、需求梳理并完成开题答辩；
第 3–4 周：完成系统总体设计、数据库 ER 图与接口说明；
第 5–10 周：分模块编码实现核心业务与权限控制；
第 11–12 周：功能测试、缺陷修复与演示数据准备；
第 13–16 周：撰写毕业论文、整理答辩 PPT 与操作说明。

七、预期成果

（1）可本地部署运行的 Web 管理系统；
（2）MySQL 建表脚本与示例数据；
（3）符合学院规范的本科毕业论文一份；
（4）系统演示视频或答辩材料。
