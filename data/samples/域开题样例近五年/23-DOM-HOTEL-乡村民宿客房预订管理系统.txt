本科毕业设计（论文）开题报告

题目：基于 Spring Boot 与 Vue 的乡村民宿客房预订管理系统的设计与实现

开题年份：2024

一、选题背景与意义

1.1 选题背景
家乡民宿在旅游旺季通过电话与微信订房，房态靠老板娘记忆，曾出现重复订房导致客人无房可住。客人询问周边餐饮加购时也无统一记录。

民宿希望有客房预订系统：展示房型与剩余房态，客人选择入住离店日期提交预订，前台确认后入住办结，并可附带房内加购小食订单（演示无真支付）。

1.2 选题意义
（1）避免超售，提升客户信任；
（2）房态与订单集中管理；
（3）加购订单与客房预订可关联；
（4）体现住宿预订业务建模能力。

二、国内外研究现状

携程、美团民宿面向平台流量，单体民宿常用轻量 PMS。本科系统做 Web 预订与演示订单即可。

三、研究目标与主要内容

3.1 研究目标
实现民宿房型/客房浏览、酒店预订选入住离店时段、订单确认、入住离店办结、可选加购及公告。

3.2 主要功能
1. 客人与前台登录；
2. 民宿房型与客房档案维护；
3. 按日期查询可预订客房；
4. 提交入住离店预订订单；
5. 前台确认预订并占用房态；
6. 入住与离店办结；
7. 可选房内加购小食订单（演示无真支付）；
8. 公告发布。

3.3 拟解决关键问题
（1）同一客房同一晚不重复预订；
（2）离店后释放房态；
（3）加购订单与主预订关联展示。

四、技术方案

后端采用 Spring Boot 构建 REST 接口，Spring Security 实现登录鉴权与角色权限；持久层使用 MyBatis 访问 MySQL 数据库。前端采用 Vue 3 与 Element Plus 搭建管理端与业务页面，ECharts 用于工作台等业务统计图表。系统按表现层、业务层、数据访问层分层设计，便于模块开发与联调。

五、非本期范围

微信支付/支付宝真实对接、OTA 渠道同步、智能门锁不在本期。

六、进度安排

第 1–2 周：文献调研、需求梳理并完成开题答辩；
第 3–4 周：完成系统总体设计、数据库 ER 图与接口说明；
第 5–10 周：分模块编码实现核心业务与权限控制；
第 11–12 周：功能测试、缺陷修复与演示数据准备；
第 13–16 周：撰写毕业论文、整理答辩 PPT 与操作说明。

七、预期成果

（1）可本地部署运行的 Web 管理系统；
（2）MySQL 建表脚本与示例数据；
（3）符合学院规范的本科毕业论文一份；
（4）系统演示视频或答辩材料。
