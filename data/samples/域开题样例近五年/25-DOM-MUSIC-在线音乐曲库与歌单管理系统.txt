本科毕业设计（论文）开题报告

题目：基于 Spring Boot 与 Vue 的在线音乐曲库与歌单管理系统的设计与实现

开题年份：2024

一、选题背景与意义

1.1 选题背景
课程设计需要展示自建曲库与歌单管理能力，以往示例仅列表播放 MP3，无法按歌手、专辑分类，用户也不能收藏喜欢的歌曲形成歌单。

拟建设在线音乐曲库系统，管理员上传或维护歌曲元数据，用户浏览曲库、外链试听并收藏到个人歌单。

1.2 选题意义
（1）提供结构化音乐曲库浏览体验；
（2）歌单收藏满足个性化需求；
（3）管理员可维护分类与推荐；
（4）训练内容型 Web 应用开发。

二、国内外研究现状

QQ 音乐、网易云商业产品非教学用途。本科毕设常见曲库 + 试听 + 收藏，不做歌词同步与转码。

三、研究目标与主要内容

3.1 研究目标
实现音乐/歌曲曲库浏览、外链试听、歌单收藏及公告管理。

3.2 主要功能
1. 用户登录；
2. 歌手、专辑、歌曲曲库档案；
3. 分类浏览与搜索；
4. 歌曲试听（外链或静态音频）；
5. 收藏到个人歌单/取消收藏；
6. 推荐歌单展示；
7. 公告发布。

3.3 拟解决关键问题
（1）收藏歌单与曲库 ID 一致；
（2）下架歌曲不可试听；
（3）检索支持歌手与曲名。

四、技术方案

后端采用 Spring Boot 构建 REST 接口，Spring Security 实现登录鉴权与角色权限；持久层使用 MyBatis 访问 MySQL 数据库。前端采用 Vue 3 与 Element Plus 搭建管理端与业务页面，ECharts 用于工作台等业务统计图表。系统按表现层、业务层、数据访问层分层设计，便于模块开发与联调。

五、非本期范围

歌词同步滚动、真转码 CDN、协同过滤推荐算法、版权付费不在本期。

六、进度安排

第 1–2 周：文献调研、需求梳理并完成开题答辩；
第 3–4 周：完成系统总体设计、数据库 ER 图与接口说明；
第 5–10 周：分模块编码实现核心业务与权限控制；
第 11–12 周：功能测试、缺陷修复与演示数据准备；
第 13–16 周：撰写毕业论文、整理答辩 PPT 与操作说明。

七、预期成果

（1）可本地部署运行的 Web 管理系统；
（2）MySQL 建表脚本与示例数据；
（3）符合学院规范的本科毕业论文一份；
（4）系统演示视频或答辩材料。
