本科毕业设计（论文）开题报告

题目：基于 Spring Boot 与 Vue 的高校会议室与球馆场地预约系统的设计与实现

开题年份：2025

一、选题背景与意义

1.1 选题背景
学院行政与学生社团经常需要预约会议室、羽毛球场、篮球馆等场地。以往在值班室登记本上抢订，信息不同步导致「到了现场却发现已被占用」。球馆周末热门时段冲突频发。

信息化中心希望统一场地档案与预约规则，师生在线查看空闲时段、提交预约并在使用后办结，管理员可审核特殊场地。

1.2 选题意义
（1）避免场地重复预订冲突；
（2）提高场地利用率与公平性；
（3）预约记录便于后勤统计；
（4）训练资源预约类系统实现。

二、国内外研究现状

企业会议系统与体育场馆 SaaS 功能丰富，学校常自建简化版。本科系统覆盖会议室、球馆等场地预约主流程。

三、研究目标与主要内容

3.1 研究目标
实现会议室/球馆场地检索、场地预约占时段、取消与使用办结及公告管理。

3.2 主要功能
1. 师生用户登录；
2. 会议室、羽毛球馆、篮球馆等场地档案；
3. 按日期查看可预约时段；
4. 提交场地预约申请；
5. 取消预约与时段释放；
6. 使用完毕管理员或用户办结；
7. 公告与使用规范。

3.3 拟解决关键问题
（1）重叠时段不可重复预约；
（2）审核类场地支持管理员审批；
（3）预约状态全流程可查。

四、技术方案

后端采用 Spring Boot 构建 REST 接口，Spring Security 实现登录鉴权与角色权限；持久层使用 MyBatis 访问 MySQL 数据库。前端采用 Vue 3 与 Element Plus 搭建管理端与业务页面，ECharts 用于工作台等业务统计图表。系统按表现层、业务层、数据访问层分层设计，便于模块开发与联调。

五、非本期范围

人脸门禁、物联网门锁、视频会议集成不在本期。

六、进度安排

第 1–2 周：文献调研、需求梳理并完成开题答辩；
第 3–4 周：完成系统总体设计、数据库 ER 图与接口说明；
第 5–10 周：分模块编码实现核心业务与权限控制；
第 11–12 周：功能测试、缺陷修复与演示数据准备；
第 13–16 周：撰写毕业论文、整理答辩 PPT 与操作说明。

七、预期成果

（1）可本地部署运行的 Web 管理系统；
（2）MySQL 建表脚本与示例数据；
（3）符合学院规范的本科毕业论文一份；
（4）系统演示视频或答辩材料。
