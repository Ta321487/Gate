本科毕业设计（论文）开题报告

题目：基于 Spring Boot 与 Vue 的健身房私教预约管理系统的设计与实现

开题年份：2025

一、选题背景与意义

1.1 选题背景
校园周边及城市健身房普遍提供私教课程。学员需要按教练档期预约上课时间，门店则要避免同一教练、同一时段被重复预约。很多小门店仍用微信群接龙或前台纸质本记录，临时改约、爽约后难以及时释放档期，教练与前台沟通成本高。

建设私教服务预约系统，可将私教项目、可约时段、学员预约与到店办结放到线上，减少档期冲突，提升门店服务体验。本课题按本科毕设规模实现可演示的 Web 预约管理系统。

1.2 选题意义
（1）规范私教课程的预约与到店履约流程；
（2）降低前台手工排班与电话确认的工作量；
（3）便于学员查询本人预约记录；
（4）完成服务预约类信息系统的设计与实现。

二、国内外研究现状

商业健身 SaaS 常含会籍、私教课包、课程表与支付。本科课题聚焦「服务项目 + 时段预约 + 到店办结」，不展开真支付与手环硬件。

三、研究目标与主要内容

3.1 研究目标
设计并实现基于 Spring Boot 与 Vue 的健身房私教预约管理系统，支持学员与门店管理员，完成私教/课程服务浏览、分时预约、取消与到店办结、公告管理。

3.2 主要功能
1. 用户登录与个人资料维护；
2. 私教课程/健身服务项目浏览与详情；
3. 学员选择教练或项目时段提交预约；
4. 预约取消与到店履约办结；
5. 本人预约记录查询；
6. 门店公告发布与查阅；
7. 管理端预约概览。

3.3 拟解决关键问题
（1）服务时段容量占用，防止超约；
（2）预约状态在预约中、已取消、已完成之间正确变更；
（3）学员端与管理端职责清晰。

四、技术方案

Spring Boot + Spring Security + MyBatis + MySQL；Vue 3 + Element Plus。REST 接口与角色菜单分离。

五、非本期范围

微信支付/支付宝真实收款、手环开门、体测仪器对接、私教课时包复杂扣次引擎不在本期实现。

六、进度安排

第 1–2 周：调研门店预约流程并开题；
第 3–4 周：完成设计文档与库表；
第 5–10 周：实现服务目录与预约办结；
第 11–12 周：测试与演示；
第 13–16 周：论文与答辩。

七、预期成果

可演示 Web 系统、数据库脚本、毕业论文与答辩材料。
