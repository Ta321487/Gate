本科毕业设计（论文）开题报告

题目：基于 Spring Boot 与 Vue 的个人博客资讯发布管理系统的设计与实现

开题年份：2025

一、选题背景与意义

1.1 选题背景
个人技术博客是展示学习成果的重要方式，以往使用第三方平台受限于主题与导出。希望自建博客系统，可发布文章与资讯、分类归档，读者浏览并收藏感兴趣的文章。

与论坛不同，博客以单向发布为主，互动仅保留收藏，不做楼中楼回帖，避免 scope 膨胀。

1.2 选题意义
（1）自主可控的内容发布与归档；
（2）文章收藏便于读者稍后阅读；
（3）练手 CMS 类前后端分离项目；
（4）为求职提供可展示作品站点。

二、国内外研究现状

WordPress、Hexo 等方案成熟，本科毕设仍常做 Spring Boot + Vue 自建 CMS 以练技术栈。

三、研究目标与主要内容

3.1 研究目标
实现博客文章/资讯发布与浏览、分类管理、文章收藏及公告。

3.2 主要功能
1. 博主与访客登录；
2. 文章分类与标签；
3. 博客文章/资讯发布与编辑；
4. 前台浏览与检索；
5. 读者收藏文章；
6. 个人收藏列表；
7. 公告发布。

3.3 拟解决关键问题
（1）草稿与已发布状态区分；
（2）收藏与文章一一对应；
（3）分类检索准确。

四、技术方案

后端采用 Spring Boot 构建 REST 接口，Spring Security 实现登录鉴权与角色权限；持久层使用 MyBatis 访问 MySQL 数据库。前端采用 Vue 3 与 Element Plus 搭建管理端与业务页面，ECharts 用于工作台等业务统计图表。系统按表现层、业务层、数据访问层分层设计，便于模块开发与联调。

五、非本期范围

富文本多人协同编辑、SEO/CDN 全球分发、评论楼中楼（属论坛域）不在本期。

六、进度安排

第 1–2 周：文献调研、需求梳理并完成开题答辩；
第 3–4 周：完成系统总体设计、数据库 ER 图与接口说明；
第 5–10 周：分模块编码实现核心业务与权限控制；
第 11–12 周：功能测试、缺陷修复与演示数据准备；
第 13–16 周：撰写毕业论文、整理答辩 PPT 与操作说明。

七、预期成果

（1）可本地部署运行的 Web 管理系统；
（2）MySQL 建表脚本与示例数据；
（3）符合学院规范的本科毕业论文一份；
（4）系统演示视频或答辩材料。
