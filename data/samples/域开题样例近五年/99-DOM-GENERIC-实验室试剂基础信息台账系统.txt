本科毕业设计（论文）开题报告

题目：基于 Spring Boot 与 Vue 的实验室试剂基础信息台账系统的设计与实现

开题年份：2024

一、选题背景与意义

1.1 选题背景
化学实验室试剂种类多，以往仅在纸质台账记录名称与存放位置，新入职同学查找费时，过期试剂清理缺少系统提醒。本课题仅做试剂基础信息维护，不涉及借用、买卖或预约。

管理员希望 Web 端维护试剂名称、规格、厂家、存放位置、备注等字段，支持分类与检索，便于实验室日常查账。

1.2 选题意义
（1）试剂信息集中维护，减少重复登记；
（2）分类检索提高实验准备效率；
（3）为后续扩展留接口但本期不做流转；
（4）训练纯档案 CRUD 类管理系统。

二、国内外研究现状

完整试剂管理系统含领用、借用、库存预警，本课题刻意收窄为台账 CRUD，作为通用档案壳演示。

三、研究目标与主要内容

3.1 研究目标
实现试剂档案增删改查、分类管理、检索与公告，不含借用审核、交易或时段预约。

3.2 主要功能
1. 管理员与用户登录；
2. 试剂分类维护；
3. 试剂档案新增、编辑、删除与查询；
4. 按名称、规格、存放位置检索；
5. 数据导出或打印列表（可选）；
6. 公告发布。

3.3 拟解决关键问题
（1）档案字段完整性与必填校验；
（2）分类与试剂条目关联正确；
（3）权限区分普通查询与管理员维护。

四、技术方案

后端采用 Spring Boot 构建 REST 接口，Spring Security 实现登录鉴权与角色权限；持久层使用 MyBatis 访问 MySQL 数据库。前端采用 Vue 3 与 Element Plus 搭建管理端与业务页面，ECharts 用于工作台等业务统计图表。系统按表现层、业务层、数据访问层分层设计，便于模块开发与联调。

五、非本期范围

试剂领用与借用审批、进销存出入库、实验预约占座、危化品全流程审批、第三方支付等不在本期实现；本期仅做基础信息台账维护与公告。

六、进度安排

第 1–2 周：文献调研、需求梳理并完成开题答辩；
第 3–4 周：完成系统总体设计、数据库 ER 图与接口说明；
第 5–10 周：分模块编码实现核心业务与权限控制；
第 11–12 周：功能测试、缺陷修复与演示数据准备；
第 13–16 周：撰写毕业论文、整理答辩 PPT 与操作说明。

七、预期成果

（1）可本地部署运行的 Web 管理系统；
（2）MySQL 建表脚本与示例数据；
（3）符合学院规范的本科毕业论文一份；
（4）系统演示视频或答辩材料。
