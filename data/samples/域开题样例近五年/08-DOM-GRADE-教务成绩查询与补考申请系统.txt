本科毕业设计（论文）开题报告

题目：基于 Spring Boot 与 Vue 的教务成绩查询与补考申请系统的设计与实现

开题年份：2023

一、选题背景与意义

1.1 选题背景
我校每学期期末结束后，学生需查询各科成绩，对不及格科目提交补考申请。以往成绩先在教务系统导出再张贴，补考申请填表交教务处，排队时间长，申请是否受理缺少在线反馈。

个别成绩录入错误时，学生申请成绩更正也需走纸质流程。教务老师希望有统一平台发布成绩档案、受理补考与更正申请并完成审核。

1.2 选题意义
（1）方便学生自助查询课程成绩；
（2）补考申请线上化，缩短教务处理周期；
（3）成绩更正申请留痕可审计；
（4）训练教务类审核流系统设计。

二、国内外研究现状

正方、青果等教务系统功能全面但学校统建，个人毕设不宜重复造轮子，常做「成绩查询 + 补考申请」轻量模块演示。商业系统对接学信网成本高，本科以校内演示为主。

三、研究目标与主要内容

3.1 研究目标
实现课程成绩档案查询、补考申请与审核、成绩更正申请与审核及公告管理。

3.2 主要功能
1. 学生与教务管理员登录；
2. 课程成绩档案维护与学生查询；
3. 补考申请提交（科目、学期、原因）；
4. 教务审核补考资格并反馈结果；
5. 成绩更正申请与审核；
6. 申请记录查询；
7. 公告发布。

3.3 拟解决关键问题
（1）仅对不及格或符合规则科目开放补考申请；
（2）审核状态清晰（待审/通过/驳回）；
（3）成绩档案与申请记录关联可查。

四、技术方案

后端采用 Spring Boot 构建 REST 接口，Spring Security 实现登录鉴权与角色权限；持久层使用 MyBatis 访问 MySQL 数据库。前端采用 Vue 3 与 Element Plus 搭建管理端与业务页面，ECharts 用于工作台等业务统计图表。系统按表现层、业务层、数据访问层分层设计，便于模块开发与联调。

五、非本期范围

学信网对接、复杂绩点引擎、智能排课、统考阅卷不在本期。

六、进度安排

第 1–2 周：文献调研、需求梳理并完成开题答辩；
第 3–4 周：完成系统总体设计、数据库 ER 图与接口说明；
第 5–10 周：分模块编码实现核心业务与权限控制；
第 11–12 周：功能测试、缺陷修复与演示数据准备；
第 13–16 周：撰写毕业论文、整理答辩 PPT 与操作说明。

七、预期成果

（1）可本地部署运行的 Web 管理系统；
（2）MySQL 建表脚本与示例数据；
（3）符合学院规范的本科毕业论文一份；
（4）系统演示视频或答辩材料。
