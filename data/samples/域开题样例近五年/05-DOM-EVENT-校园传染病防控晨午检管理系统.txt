本科毕业设计（论文）开题报告

题目：基于 Spring Boot 的校园传染病防控与晨午检管理系统的设计与实现

一、选题背景与意义

1.1 选题背景
学校属于人员密集场所，传染病易在班级、宿舍等微环境中传播。教育与卫生部门长期要求学校落实晨午检、因病缺勤追踪、复课查验、健康教育与应急处置等制度。实际执行中，不少学校仍采用纸质晨检表或分散表格汇总，存在数据难统计、缺勤原因难追踪、密切接触学生难管理、复课证明核验效率低、校医与班主任协同不畅等问题。

在校园数字化与学生健康管理需求增强的背景下，建设一套面向中小学或高校的传染病防控与晨午检管理系统，能够把日常监测与应急处置纳入同一平台，提高学校公共卫生管理规范化水平。

1.2 选题意义
（1）助力学校落实晨午检与因病缺勤追踪制度；
（2）形成学生健康档案、异常学生闭环管理、班级维度统计与物资保障台账；
（3）完成基于 Spring Boot 的完整软件工程训练；
（4）面向校园常态化健康管理，应用场景明确。

二、国内外研究现状

部分国家在学校健康信息系统中整合出勤、疫苗接种与传染病报告，强调家校协同与隐私保护。国内中小学普遍建立晨午检制度，相关信息化产品包括校园健康打卡与防疫管理工具。现有系统常见问题是：与日常教务协同不足、统计分析弱、权限角色不清晰、难复用到流感季等常规传染病场景。

三、研究目标与主要内容

3.1 研究目标
设计并实现基于 Spring Boot 的校园传染病防控与晨午检管理系统，支持管理员、校医、班主任、学生（或家长代填）等角色，完成日常晨午检、异常学生跟踪、密接同学管理、复课登记、物资库存与可视化统计。

3.2 主要内容
（1）调研学校晨午检与传染病报告流程；
（2）完成系统架构、功能结构、数据库与安全设计；
（3）实现核心业务模块与权限控制；
（4）进行测试并撰写论文。

3.3 拟解决关键问题
（1）晨午检高频填报与异常自动进入重点关注列表；
（2）班级—宿舍维度的接触者快速登记与解除；
（3）复课查验与解除观察的状态闭环；
（4）校医全局视角与班主任班级视角的数据权限隔离。

四、技术方案与系统架构

4.1 技术选型
Spring Boot、Spring Security、MyBatis、MySQL、Thymeleaf、AdminLTE、jQuery、ECharts、PageHelper、Druid、Maven。

4.2 逻辑架构
表现层 → 业务层（晨检、学生健康、排查、复课、物资）→ 持久层 → MySQL。

4.3 安全方案
Spring Security 认证授权；按角色显示菜单；通过班主任绑定班级实现简化版数据权限。

五、系统功能模块设计

5.1 校园健康驾驶舱
今日晨检应到/实到、发热异常人数、在管异常学生、今日复课人数、物资预警；按年级班级统计。

5.2 用户与权限管理
系统管理员、校医、班主任、普通用户。

5.3 晨午检打卡
学生每日提交体温、症状、是否请假；班主任可代填；异常记录高亮。

5.4 异常学生管理
对发热、疑似、确诊传染病的学生建档，记录病种、发病日期、隔离方式、就诊情况。

5.5 密切接触学生管理
登记同桌、同宿舍、同小组等接触者，设置观察期，支持转异常或解除。

5.6 复课与转归管理
复课登记、痊愈返校、仍在治疗等状态管理。

5.7 检测与查验记录
登记检测结果或医院证明查验，支持复课前核验。

5.8 校园防疫物资管理
体温枪、口罩、消毒液等出入库与库存预警。

5.9 专题页面
检测查验专题、物资保障专题等。

六、数据库设计

用户与权限、班级、学生基础信息、晨午检记录、异常学生、密接学生、查验记录、复课/转归、物资与出入库。关键状态建议：正常、异常观察、确诊在管、复课、解除。

七、可行性分析

Spring Boot 技术成熟；校园场景清晰；使用虚构学生数据，论文声明隐私保护。

八、进度安排

第1–2周：选题、文献、开题；
第3–4周：需求规格、用例、库表设计；
第5–8周：权限、学生档案、晨午检；
第9–11周：异常学生、密接、复课、物资、看板；
第12–13周：测试与优化；
第14–16周：论文、PPT、答辩演练。

九、预期成果

可演示系统、数据库脚本、操作说明、毕业论文、测试记录与答辩材料。

十、创新点与特色

（1）校园常态化健康管理闭环；
（2）晨午检高频数据与异常处置联动；
（3）班级数据权限体现校园管理特点；
（4）复课查验流程信息化。

十一、难点与对策

角色较多：先实现管理员+校医+班主任；
病种复杂：用病种字典简化；
移动端需求：以 Web 为主，展望中写小程序扩展。
