本科毕业设计（论文）开题报告

题目：基于 Spring Boot 与 Vue 的中小企业客户跟进管理系统的设计与实现

开题年份：2025

一、选题背景与意义

1.1 选题背景
学院创业孵化中心有多家学生初创团队，销售环节依赖个人微信与表格记录客户线索。成员轮换后历史沟通记录丢失，主管无法掌握每条线索跟进到了哪一步，容易遗漏重点客户回访，成交转化率难以统计。

通用 CRM 软件按账号收费且功能冗余，小团队更需要轻量工具：录入客户、记录每次跟进、标记阶段并在审核后归档。本课题拟为中小团队做一套可自部署的客户跟进管理系统。

1.2 选题意义
（1）避免客户线索断档，提升跟进连续性；
（2）主管可查看跟进进度与完结情况；
（3）沉淀客户档案，支持简单统计分析；
（4）训练 CRM 业务建模与权限设计能力。

二、国内外研究现状

Salesforce、纷享销客等商业 CRM 功能完善但价格与学习成本高。国内不少毕设选择简化版客户管理，仅保留档案、跟进记录与审核完结。本系统定位轻量 Web，满足校园创业团队演示需求。

三、研究目标与主要内容

3.1 研究目标
实现客户档案分类、跟进记录提交、管理员审核完结、客户阶段管理与公告功能。

3.2 主要功能
1. 销售/运营人员登录与个人资料；
2. 客户档案分类录入与检索；
3. 客户跟进记录提交（沟通方式、意向、下次计划）；
4. 主管审核跟进并标记完结或继续跟进；
5. 客户列表按阶段筛选与统计；
6. 公告与团队通知。

3.3 拟解决关键问题
（1）客户跟进状态流转清晰，避免重复完结；
（2）敏感客户信息按角色控制可见范围；
（3）跟进历史可追溯，支持答辩演示完整链路。

四、技术方案

后端采用 Spring Boot 构建 REST 接口，Spring Security 实现登录鉴权与角色权限；持久层使用 MyBatis 访问 MySQL 数据库。前端采用 Vue 3 与 Element Plus 搭建管理端与业务页面，ECharts 用于工作台等业务统计图表。系统按表现层、业务层、数据访问层分层设计，便于模块开发与联调。

五、非本期范围

外呼中心、公海池自动分配、邮件营销、与企微深度集成不在本期。

六、进度安排

第 1–2 周：文献调研、需求梳理并完成开题答辩；
第 3–4 周：完成系统总体设计、数据库 ER 图与接口说明；
第 5–10 周：分模块编码实现核心业务与权限控制；
第 11–12 周：功能测试、缺陷修复与演示数据准备；
第 13–16 周：撰写毕业论文、整理答辩 PPT 与操作说明。

七、预期成果

（1）可本地部署运行的 Web 管理系统；
（2）MySQL 建表脚本与示例数据；
（3）符合学院规范的本科毕业论文一份；
（4）系统演示视频或答辩材料。
