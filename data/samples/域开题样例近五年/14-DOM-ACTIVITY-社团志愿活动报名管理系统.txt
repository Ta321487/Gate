本科毕业设计（论文）开题报告

题目：基于 Spring Boot 与 Vue 的社团志愿活动报名管理系统的设计与实现

开题年份：2024

一、选题背景与意义

1.1 选题背景
我校青年志愿者协会每学期组织多次志愿服务与社团活动，报名以往通过群接龙或现场报名，活动名额有限时难以公平排序，且无法自动检测学生已报的其他活动是否时间冲突。

活动负责人还需人工统计签到与审核报名材料。建设社团志愿活动报名管理系统，支持活动发布、报名占名额、时段冲突提示与审核。

1.2 选题意义
（1）提高活动组织效率与名额公平性；
（2）报名与审核线上留痕；
（3）时段冲突提示减少学生误报；
（4）支撑校园第二课堂信息化实践。

二、国内外研究现状

活动发布平台较多，但常与第二课堂学分系统绑定。本科独立系统聚焦活动档案、报名审核、名额占用与冲突检测。

三、研究目标与主要内容

3.1 研究目标
实现社团活动/志愿活动发布、学生报名申请、名额占用与审核、时段冲突提示及公告管理。

3.2 主要功能
1. 学生、社团管理员登录；
2. 活动分类与活动档案发布（时间、地点、名额）；
3. 学生提交活动报名申请；
4. 管理员审核通过并占用名额；
5. 本人已报活动时段冲突提示；
6. 报名记录与活动名单查询；
7. 公告发布。

3.3 拟解决关键问题
（1）名额满后停止接受新报名；
（2）同一时段多活动冲突检测；
（3）审核通过后生成活动参与名单。

四、技术方案

后端采用 Spring Boot 构建 REST 接口，Spring Security 实现登录鉴权与角色权限；持久层使用 MyBatis 访问 MySQL 数据库。前端采用 Vue 3 与 Element Plus 搭建管理端与业务页面，ECharts 用于工作台等业务统计图表。系统按表现层、业务层、数据访问层分层设计，便于模块开发与联调。

五、非本期范围

人脸签到硬件、GPS 定位签到、学分自动认定对接不在本期。

六、进度安排

第 1–2 周：文献调研、需求梳理并完成开题答辩；
第 3–4 周：完成系统总体设计、数据库 ER 图与接口说明；
第 5–10 周：分模块编码实现核心业务与权限控制；
第 11–12 周：功能测试、缺陷修复与演示数据准备；
第 13–16 周：撰写毕业论文、整理答辩 PPT 与操作说明。

七、预期成果

（1）可本地部署运行的 Web 管理系统；
（2）MySQL 建表脚本与示例数据；
（3）符合学院规范的本科毕业论文一份；
（4）系统演示视频或答辩材料。
