本科毕业设计（论文）开题报告

题目：基于 Spring Boot 与 Vue 的高校食堂档口点餐管理系统的设计与实现

开题年份：2025

一、选题背景与意义

1.1 选题背景
我校第一食堂午高峰排队时间长，学生希望提前在手机选好菜品，到档口报取餐码直接取餐。目前档口仅支持现场点单，厨师难以预估备餐量，造成浪费或供应不足。

各档口菜品与套餐价格更新频繁，纸质菜单更换不及时。需要食堂档口点餐管理系统支持菜品浏览、点餐下单、档口接单与取餐核销。

1.2 选题意义
（1）缩短高峰排队时间；
（2）档口可提前备餐，减少浪费；
（3）订单记录便于统计热销菜品；
（4）体现 O2O 点餐场景开发能力。

二、国内外研究现状

美团、饿了么面向校外配送，校内食堂常自建点餐小程序。本科 Web 版聚焦点餐下单与取餐，不做骑手调度与真支付。

三、研究目标与主要内容

3.1 研究目标
实现菜品/套餐浏览、点餐下单、食堂档口接单、取餐与订单查询及留言公告。

3.2 主要功能
1. 师生用户登录；
2. 食堂档口与菜品/套餐档案维护；
3. 浏览菜单并提交点餐订单；
4. 档口管理员接单与备餐状态更新；
5. 取餐码核销与订单完成；
6. 订单历史查询；
7. 访客留言与公告。

3.3 拟解决关键问题
（1）下单后档口可见待制作订单；
（2）取餐核销后订单完结；
（3）高峰时段库存或限售控制（简化）。

四、技术方案

后端采用 Spring Boot 构建 REST 接口，Spring Security 实现登录鉴权与角色权限；持久层使用 MyBatis 访问 MySQL 数据库。前端采用 Vue 3 与 Element Plus 搭建管理端与业务页面，ECharts 用于工作台等业务统计图表。系统按表现层、业务层、数据访问层分层设计，便于模块开发与联调。

五、非本期范围

真支付、外卖骑手调度、食品安全追溯大屏不在本期；勿与食品安全风险排查系统混淆。

六、进度安排

第 1–2 周：文献调研、需求梳理并完成开题答辩；
第 3–4 周：完成系统总体设计、数据库 ER 图与接口说明；
第 5–10 周：分模块编码实现核心业务与权限控制；
第 11–12 周：功能测试、缺陷修复与演示数据准备；
第 13–16 周：撰写毕业论文、整理答辩 PPT 与操作说明。

七、预期成果

（1）可本地部署运行的 Web 管理系统；
（2）MySQL 建表脚本与示例数据；
（3）符合学院规范的本科毕业论文一份；
（4）系统演示视频或答辩材料。
