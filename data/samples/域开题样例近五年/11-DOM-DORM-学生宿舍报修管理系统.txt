本科毕业设计（论文）开题报告

题目：基于 Spring Boot 与 Vue 的学生宿舍报修管理系统的设计与实现

开题年份：2024

一、选题背景与意义

1.1 选题背景
学生宿舍水管漏水、灯具损坏、空调不制冷等问题几乎每天都有。以往学生到宿管值班室口头报修，维修工排期不透明，同一问题多次报修因无工单号难以追踪，维修完成后也缺少确认环节。

期末与雨季报修量激增，宿管中心希望学生可上传现场照片、选择故障类型，管理员派单给维修人员并在系统内完结。

1.2 选题意义
（1）缩短宿舍故障响应时间；
（2）报修工单全程可追溯；
（3）维修派单与完结确认规范化；
（4）体现工单类管理系统实现能力。

二、国内外研究现状

智慧校园平台常含报修模块，也可独立开发。商业物业系统功能重。本科 Web 系统聚焦提交报修、派单处理、完结回访即可。

三、研究目标与主要内容

3.1 研究目标
实现宿舍报修工单提交、管理员派单处理、维修完结与公告管理。

3.2 主要功能
1. 学生登录、宿舍号与个人资料；
2. 报修工单提交（类型、描述、附件照片）；
3. 管理员受理、派单与处理进度更新；
4. 维修完结与学生确认；
5. 报修历史查询；
6. 公告发布。

3.3 拟解决关键问题
（1）工单状态流转（待受理/处理中/已完结）；
（2）附件上传便于维修人员预判；
（3）按宿舍楼或片区分配处理人（简化版）。

四、技术方案

后端采用 Spring Boot 构建 REST 接口，Spring Security 实现登录鉴权与角色权限；持久层使用 MyBatis 访问 MySQL 数据库。前端采用 Vue 3 与 Element Plus 搭建管理端与业务页面，ECharts 用于工作台等业务统计图表。系统按表现层、业务层、数据访问层分层设计，便于模块开发与联调。

五、非本期范围

人脸门禁、物联网水电表自动报障、智能锁联动不在本期。

六、进度安排

第 1–2 周：文献调研、需求梳理并完成开题答辩；
第 3–4 周：完成系统总体设计、数据库 ER 图与接口说明；
第 5–10 周：分模块编码实现核心业务与权限控制；
第 11–12 周：功能测试、缺陷修复与演示数据准备；
第 13–16 周：撰写毕业论文、整理答辩 PPT 与操作说明。

七、预期成果

（1）可本地部署运行的 Web 管理系统；
（2）MySQL 建表脚本与示例数据；
（3）符合学院规范的本科毕业论文一份；
（4）系统演示视频或答辩材料。
