本科毕业设计（论文）开题报告

题目：基于 Spring Boot 与 Vue 的高校图书借阅管理系统的设计与实现

开题年份：2024

一、选题背景与意义

1.1 选题背景
我校图书馆现有纸质藏书十余万册，读者以本科生和研究生为主。期末复习与论文撰写期间，借还高峰集中在服务台，工作人员需手工登记借阅证号、核对索书号并计算应还日期，排队时间长且容易录错。部分读者不清楚馆藏位置与可借册数，重复咨询占用了大量人力。

馆员每日还要电话或微信群催促逾期读者，逾期台账与在架状态往往不同步，导致「系统显示在借、架上却有书」的差错。建设一套面向读者的线上借阅管理系统，可让检索、借阅申请、审核与归还闭环在 Web 端完成，减轻窗口压力。

1.2 选题意义
（1）规范图书借阅与归还流程，提高馆藏周转效率；
（2）为读者提供检索、预约借阅与逾期提醒等自助服务；
（3）沉淀借阅统计数据，辅助采购与剔旧决策；
（4）完成前后端分离 Web 系统的完整工程实践。

二、国内外研究现状

国外高校普遍采用集成图书馆系统（ILS），功能完善但部署与定制成本高。国内商业 ILS 面向大型馆，高职本科自建系统多聚焦借还审核、馆藏检索等核心环节。本科毕设适合实现轻量 Web 版，满足教学演示与流程数字化即可。

三、研究目标与主要内容

3.1 研究目标
设计并实现基于 Spring Boot 与 Vue 的高校图书借阅管理系统，支持管理员、读者等角色，完成图书档案维护、借阅申请与审核、归还登记、应还日期与逾期提醒、公告发布及基础统计。

3.2 主要功能
1. 用户注册登录、个人资料与读者证信息维护；
2. 图书分类与书目档案检索、详情查看；
3. 借阅申请提交与管理员审核，占用可借册数；
4. 归还登记、应还日期计算与逾期提醒；
5. 借阅记录查询与借阅量统计图表；
6. 系统公告与推荐书目展示。

3.3 拟解决关键问题
（1）借阅审核通过后准确占用馆藏并生成应还日期；
（2）逾期状态与在架/在借标识保持一致；
（3）按角色区分馆员审核与读者自助查询权限。

四、技术方案

后端采用 Spring Boot 构建 REST 接口，Spring Security 实现登录鉴权与角色权限；持久层使用 MyBatis 访问 MySQL 数据库。前端采用 Vue 3 与 Element Plus 搭建管理端与业务页面，ECharts 用于工作台等业务统计图表。系统按表现层、业务层、数据访问层分层设计，便于模块开发与联调。

五、非本期范围

人脸识别进馆、RFID 自助借还、协同过滤推荐、与 CALIS 联合编目对接等不在本期实现；系统以软件流程演示为主。

六、进度安排

第 1–2 周：文献调研、需求梳理并完成开题答辩；
第 3–4 周：完成系统总体设计、数据库 ER 图与接口说明；
第 5–10 周：分模块编码实现核心业务与权限控制；
第 11–12 周：功能测试、缺陷修复与演示数据准备；
第 13–16 周：撰写毕业论文、整理答辩 PPT 与操作说明。

七、预期成果

（1）可本地部署运行的 Web 管理系统；
（2）MySQL 建表脚本与示例数据；
（3）符合学院规范的本科毕业论文一份；
（4）系统演示视频或答辩材料。
