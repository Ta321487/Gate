本科毕业设计（论文）开题报告

题目：基于 Spring Boot 与 Vue 的学生请假销假管理系统的设计与实现

开题年份：2024

一、选题背景与意义

1.1 选题背景
我校学工规定学生因病、因事请假须提前提交申请，辅导员审批后方可离校。以往学生填纸质假条或发微信截图，辅导员难以统一存档，销假环节依赖返校后口头确认，缺勤统计与请假台账对不上。

期末与实习离校期间请假集中，人工审批效率低且不易追溯。需要一套请假销假管理系统，实现在线申请、多级审批、销假完结与假勤查询。

1.2 选题意义
（1）规范请销假流程，减少虚假请假；
（2）审批留痕，便于学工核查；
（3）销假完结后自动更新假勤状态；
（4）体现工作流类管理系统设计能力。

二、国内外研究现状

中小学智慧校园产品多含考勤请假模块，高校也有教务或学工系统覆盖。商业方案常与人脸闸机绑定。本科毕设实现 Web 版请假申请、审批与销假即可，无需硬件考勤。

三、研究目标与主要内容

3.1 研究目标
实现学生请假申请、辅导员/管理员审批、销假完结、假勤台账查询及公告管理。

3.2 主要功能
1. 学生登录与个人资料、班级信息；
2. 人员档案维护（辅导员绑定班级）；
3. 请假申请（事假、病假、时段与事由）；
4. 审批通过/驳回与审批意见；
5. 返校销假确认与假单完结；
6. 假勤记录查询与统计；
7. 公告发布。

3.3 拟解决关键问题
（1）请假时段与重复请假校验；
（2）未销假假单保持进行中状态并提醒；
（3）辅导员仅审批本班学生申请。

四、技术方案

后端采用 Spring Boot 构建 REST 接口，Spring Security 实现登录鉴权与角色权限；持久层使用 MyBatis 访问 MySQL 数据库。前端采用 Vue 3 与 Element Plus 搭建管理端与业务页面，ECharts 用于工作台等业务统计图表。系统按表现层、业务层、数据访问层分层设计，便于模块开发与联调。

五、非本期范围

人脸考勤、GPS 轨迹打卡、与门禁闸机联动不在本期。

六、进度安排

第 1–2 周：文献调研、需求梳理并完成开题答辩；
第 3–4 周：完成系统总体设计、数据库 ER 图与接口说明；
第 5–10 周：分模块编码实现核心业务与权限控制；
第 11–12 周：功能测试、缺陷修复与演示数据准备；
第 13–16 周：撰写毕业论文、整理答辩 PPT 与操作说明。

七、预期成果

（1）可本地部署运行的 Web 管理系统；
（2）MySQL 建表脚本与示例数据；
（3）符合学院规范的本科毕业论文一份；
（4）系统演示视频或答辩材料。
