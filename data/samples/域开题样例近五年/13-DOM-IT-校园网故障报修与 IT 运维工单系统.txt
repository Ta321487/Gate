本科毕业设计（论文）开题报告

题目：基于 Spring Boot 与 Vue 的校园网故障报修与 IT 运维工单系统的设计与实现

开题年份：2023

一、选题背景与意义

1.1 选题背景
我校网络中心负责宿舍区校园网与部分机房终端维护。师生遇到无法上网、认证失败、交换机端口故障等问题，常通过电话或到值班室登记，值班同学记录本易丢，重复故障无法统计高发区域。

机房管理员也希望老师报修投影、电脑故障时有工单编号可跟踪。建设 IT 运维报修系统，可分类受理网络报修与机房报修，运维人员处理并完结。

1.2 选题意义
（1）统一 IT 故障受理入口；
（2）工单处理时效可统计；
（3）常见故障知识可公告沉淀；
（4）锻炼运维工单系统开发经验。

二、国内外研究现状

企业 ITSM 系统功能全面，高校多自建轻量报修页。本科系统不做 SNMP 采集与自动拓扑，仅软件工单流。

三、研究目标与主要内容

3.1 研究目标
实现网络报修、机房报修工单提交，IT 运维审核处理、完结及公告管理。

3.2 主要功能
1. 师生用户登录；
2. 网络报修工单（地点、现象、联系方式）；
3. 机房/多媒体教室设备报修工单；
4. IT 运维受理、分派与处理记录；
5. 工单完结与历史查询；
6. 公告与网络维护通知。

3.3 拟解决关键问题
（1）区分网络类与终端类工单便于分派；
（2）处理过程留痕支持交接班；
（3）超时未处理工单列表提醒。

四、技术方案

后端采用 Spring Boot 构建 REST 接口，Spring Security 实现登录鉴权与角色权限；持久层使用 MyBatis 访问 MySQL 数据库。前端采用 Vue 3 与 Element Plus 搭建管理端与业务页面，ECharts 用于工作台等业务统计图表。系统按表现层、业务层、数据访问层分层设计，便于模块开发与联调。

五、非本期范围

网络设备 SNMP 采集、自动拓扑发现、远程桌面控制不在本期。

六、进度安排

第 1–2 周：文献调研、需求梳理并完成开题答辩；
第 3–4 周：完成系统总体设计、数据库 ER 图与接口说明；
第 5–10 周：分模块编码实现核心业务与权限控制；
第 11–12 周：功能测试、缺陷修复与演示数据准备；
第 13–16 周：撰写毕业论文、整理答辩 PPT 与操作说明。

七、预期成果

（1）可本地部署运行的 Web 管理系统；
（2）MySQL 建表脚本与示例数据；
（3）符合学院规范的本科毕业论文一份；
（4）系统演示视频或答辩材料。
