本科毕业设计（论文）开题报告

题目：基于 Spring Boot 与 Vue 的宠物医院挂号预约管理系统的设计与实现

开题年份：2025

一、选题背景与意义

1.1 选题背景
宠物医疗需求持续增长，中小型宠物医院门诊高峰时段常出现窗口排队挂号、主人携带宠物等候时间过长的情况。仅靠电话预约或前台口头登记，容易出现号源冲突、爽约后号源未及时释放、到诊顺序不清等问题。

建设面向宠物医院的挂号预约管理系统，可让宠主按科室或医师查看可约时段并提交预约，医院侧完成就诊办结或取消，从而缓解窗口压力。本课题定位为门诊挂号预约，不以电子病历、处方开药为主线。

1.2 选题意义
（1）改善宠主挂号体验，缩短无效等候；
（2）规范号源时段占用与释放；
（3）便于医院查询当日预约与履约情况；
（4）完成医疗轻量预约类 Web 系统实践。

二、国内外研究现状

国外宠物医院信息系统（PIMS）功能较全；国内连锁门店多采购商业系统，单体诊所信息化程度参差。本科毕设适合实现挂号预约闭环演示，不扩展完整电子病历与检验流程。

三、研究目标与主要内容

3.1 研究目标
设计并实现基于 Spring Boot 与 Vue 的宠物医院挂号预约管理系统，支持宠主与医院管理员，完成科室/号源浏览、分时挂号预约、取消与就诊办结、公告管理。

3.2 主要功能
1. 用户登录与个人资料维护；
2. 科室或医师号源、可约时段浏览；
3. 提交挂号预约（可填写就诊宠物等信息）；
4. 预约取消与就诊办结；
5. 本人预约记录查询；
6. 院内公告发布与查阅；
7. 管理端预约概览。

3.3 拟解决关键问题
（1）同一号源时段容量控制，避免超约；
（2）预约与办结状态一致可查；
（3）前台管理与宠主自助权限分离。

四、技术方案

Spring Boot、Spring Security、MyBatis、MySQL；前端 Vue 3、Element Plus。分层 B/S 架构。

五、非本期范围

电子病历全文、处方开药、检验检查闭环、排队叫号大屏、医保结算等不在本期实现。病患长期随访与隔离观察另题讨论，不作为本系统主路径。

六、进度安排

第 1–2 周：调研宠物医院挂号流程并开题；
第 3–4 周：需求分析与数据库设计；
第 5–10 周：号源与预约模块开发；
第 11–12 周：测试与演示数据；
第 13–16 周：论文与答辩。

七、预期成果

可运行系统、数据库脚本、毕业论文与答辩材料。
