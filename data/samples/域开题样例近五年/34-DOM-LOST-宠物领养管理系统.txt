本科毕业设计（论文）开题报告

题目：基于 Spring Boot 与 Vue 的宠物领养管理系统的设计与实现

开题年份：2025

一、选题背景与意义

1.1 选题背景
城市流浪动物救助与宠物领养需求并存。救助站或校园爱心社团常通过社交平台发布待领养动物信息，意向领养人私信咨询后，仍需人工核对家庭条件、签署协议并安排送养。信息分散在多个群聊中，容易出现重复咨询、领养进度不清、已领养动物未及时下架等问题。

建设宠物领养管理系统，可发布待领养动物档案，由意向领养人提交领养申请，管理员审核通过或驳回并办结，从而使领养流程可追溯。本课题以信息发布与申请审核为主线。

1.2 选题意义
（1）提高待领养信息的组织与检索效率；
（2）规范领养申请与审核留痕；
（3）减少社交群聊中的信息错漏；
（4）完成申请审核类 Web 系统的工程训练。

二、国内外研究现状

国外宠物领养平台注重匹配与志愿者协同；国内以公益组织站点与商业宠物平台为主。本科毕设实现「动物档案 + 领养申请审核」即可，不扩展影像识别与电商支付。

三、研究目标与主要内容

3.1 研究目标
设计并实现基于 Spring Boot 与 Vue 的宠物领养管理系统，支持意向领养人与管理员，完成待领养动物信息发布、领养申请提交与审核办结、公告管理。

3.2 主要功能
1. 用户登录与个人资料维护；
2. 待领养宠物档案分类浏览与详情；
3. 提交领养申请（可说明饲养条件等）；
4. 管理员审核通过或驳回，办结申请；
5. 本人申请记录查询；
6. 领养须知与活动公告管理；
7. 管理端待审与办结概览。

3.3 拟解决关键问题
（1）动物档案与领养申请正确关联；
（2）审核状态清晰，避免重复有效申请冲突；
（3）普通用户与管理员权限分离。

四、技术方案

Spring Boot、Spring Security、MyBatis、MySQL；前端 Vue 3、Element Plus。

五、非本期范围

宠物影像智能识别匹配、真支付保证金、活体物流轨迹、小程序原生交付不在本期实现。

六、进度安排

第 1–2 周：调研领养流程并开题；
第 3–4 周：需求与数据库设计；
第 5–10 周：档案与申请审核开发；
第 11–12 周：测试与演示；
第 13–16 周：论文与答辩。

七、预期成果

可运行系统、数据库脚本、毕业论文与答辩材料。
