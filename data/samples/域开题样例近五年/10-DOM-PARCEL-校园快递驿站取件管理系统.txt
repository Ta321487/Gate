本科毕业设计（论文）开题报告

题目：基于 Spring Boot 与 Vue 的校园快递驿站取件管理系统的设计与实现

开题年份：2023

一、选题背景与意义

1.1 选题背景
我校南门快递驿站日均入库包裹上千件，高峰期货架堆积，工作人员扫码入库后短信通知取件码，但部分学生拖延取件占用货位。手工台账难以统计滞留件，催取靠张贴告示效率低。

取件核销时偶尔出现冒领，需核对取件码与身份。建设校园快递驿站取件管理系统，可规范入库建档、取件码生成、取件申请核销与库存占用释放。

1.2 选题意义
（1）提高驿站周转率，缓解货位紧张；
（2）取件码与核销留痕，降低冒领风险；
（3）滞留包裹可统计与催取；
（4）训练物流末端信息化场景建模。

二、国内外研究现状

菜鸟驿站、丰巢等商业方案含智能柜与支付，本科毕设适合 Web 版入库取件流程，不对接柜机硬件。

三、研究目标与主要内容

3.1 研究目标
实现包裹入库建档、取件码分配、取件核销、库存占用与催取公告。

3.2 主要功能
1. 用户登录（收件人、驿站管理员）；
2. 包裹入库登记与取件码生成；
3. 收件人查询待取包裹；
4. 取件核销并释放货位占用；
5. 滞留件统计与催取提醒；
6. 公告发布。

3.3 拟解决关键问题
（1）取件码与包裹一一对应且核销后失效；
（2）已取件包裹从待取列表移除；
（3）入库数量与在库占用数量一致。

四、技术方案

后端采用 Spring Boot 构建 REST 接口，Spring Security 实现登录鉴权与角色权限；持久层使用 MyBatis 访问 MySQL 数据库。前端采用 Vue 3 与 Element Plus 搭建管理端与业务页面，ECharts 用于工作台等业务统计图表。系统按表现层、业务层、数据访问层分层设计，便于模块开发与联调。

五、非本期范围

智能快递柜硬件对接、运费到付支付、与快递公司总部 API 对接不在本期。

六、进度安排

第 1–2 周：文献调研、需求梳理并完成开题答辩；
第 3–4 周：完成系统总体设计、数据库 ER 图与接口说明；
第 5–10 周：分模块编码实现核心业务与权限控制；
第 11–12 周：功能测试、缺陷修复与演示数据准备；
第 13–16 周：撰写毕业论文、整理答辩 PPT 与操作说明。

七、预期成果

（1）可本地部署运行的 Web 管理系统；
（2）MySQL 建表脚本与示例数据；
（3）符合学院规范的本科毕业论文一份；
（4）系统演示视频或答辩材料。
