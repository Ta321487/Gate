本科毕业设计（论文）开题报告

题目：基于 Spring Boot 与 Vue 的校园影视资源点播管理系统的设计与实现

开题年份：2023

一、选题背景与意义

1.1 选题背景
我校传媒学院为教学与社团展映积累了大量正版授权影视资源，以往通过共享文件夹分发，检索不便且缺少观看记录。学生希望按电影、电视剧、综艺分类浏览并收藏片单。

系统不对视频做转码，可外链或校内静态资源播放，重点在片库档案、分类检索与收藏。

1.2 选题意义
（1）规范校内影视资源展示与检索；
（2）收藏片单便于课程与社团使用；
（3）管理员可维护分类与上下架；
（4）训练媒资内容管理系统开发。

二、国内外研究现状

爱奇艺、B 站等综合视频平台非校内专库。本科系统做片库浏览、播放跳转与收藏，不做 CDN 转码。

三、研究目标与主要内容

3.1 研究目标
实现电影/电视剧/综艺片库浏览、外链播放、片单收藏及公告管理。

3.2 主要功能
1. 用户登录；
2. 影视分类与片库档案维护；
3. 按片名、类型检索与详情；
4. 点播播放（外链或本地静态文件）；
5. 个人片单收藏与取消；
6. 热门片单推荐（简化列表）；
7. 公告发布。

3.3 拟解决关键问题
（1）下架片源不可继续播放；
（2）收藏与片库条目关联正确；
（3）分类检索性能满足演示规模。

四、技术方案

后端采用 Spring Boot 构建 REST 接口，Spring Security 实现登录鉴权与角色权限；持久层使用 MyBatis 访问 MySQL 数据库。前端采用 Vue 3 与 Element Plus 搭建管理端与业务页面，ECharts 用于工作台等业务统计图表。系统按表现层、业务层、数据访问层分层设计，便于模块开发与联调。

五、非本期范围

直播弹幕、视频转码 CDN、影院选座购票、版权 DRM 不在本期。

六、进度安排

第 1–2 周：文献调研、需求梳理并完成开题答辩；
第 3–4 周：完成系统总体设计、数据库 ER 图与接口说明；
第 5–10 周：分模块编码实现核心业务与权限控制；
第 11–12 周：功能测试、缺陷修复与演示数据准备；
第 13–16 周：撰写毕业论文、整理答辩 PPT 与操作说明。

七、预期成果

（1）可本地部署运行的 Web 管理系统；
（2）MySQL 建表脚本与示例数据；
（3）符合学院规范的本科毕业论文一份；
（4）系统演示视频或答辩材料。
