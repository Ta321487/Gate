本科毕业设计（论文）开题报告

题目：基于 Spring Boot 与 Vue 的校医院门诊挂号预约系统的设计与实现

开题年份：2023

一、选题背景与意义

1.1 选题背景
我校校医院上午门诊常出现排队挂号现象，学生不知道各科室剩余号源，现场咨询占用了大量导诊时间。感冒季内科号源紧张，未预约学生集中到达进一步加剧拥堵。

校医院希望提供分时挂号预约：学生在线查看科室与号源时段，预约后按时段到院，就诊完成后由管理员办结。不涉及电子病历与处方。

1.2 选题意义
（1）分流挂号高峰，改善就医体验；
（2）号源占用清晰，减少超约；
（3）预约记录可统计科室负载；
（4）训练医疗预约类系统设计（轻量）。

二、国内外研究现状

三甲医院挂号系统功能完整含医保结算，校医院本科毕设聚焦科室号源、挂号预约、取消与就诊办结即可。

三、研究目标与主要内容

3.1 研究目标
实现科室/号源档案、门诊挂号时段预约、取消预约、就诊办结及公告管理。

3.2 主要功能
1. 师生用户登录与资料；
2. 科室与医生号源时段发布；
3. 门诊挂号预约提交；
4. 取消预约与号源释放；
5. 到院就诊后管理员办结；
6. 预约记录查询；
7. 公告与就医须知。

3.3 拟解决关键问题
（1）同一时段号源不重复预约；
（2）取消后号源及时回补；
（3）预约状态（已约/已取消/已就诊）清晰。

四、技术方案

后端采用 Spring Boot 构建 REST 接口，Spring Security 实现登录鉴权与角色权限；持久层使用 MyBatis 访问 MySQL 数据库。前端采用 Vue 3 与 Element Plus 搭建管理端与业务页面，ECharts 用于工作台等业务统计图表。系统按表现层、业务层、数据访问层分层设计，便于模块开发与联调。

五、非本期范围

电子病历、处方开药、检验检查、排队叫号大屏、医保结算不在本期。

六、进度安排

第 1–2 周：文献调研、需求梳理并完成开题答辩；
第 3–4 周：完成系统总体设计、数据库 ER 图与接口说明；
第 5–10 周：分模块编码实现核心业务与权限控制；
第 11–12 周：功能测试、缺陷修复与演示数据准备；
第 13–16 周：撰写毕业论文、整理答辩 PPT 与操作说明。

七、预期成果

（1）可本地部署运行的 Web 管理系统；
（2）MySQL 建表脚本与示例数据；
（3）符合学院规范的本科毕业论文一份；
（4）系统演示视频或答辩材料。
