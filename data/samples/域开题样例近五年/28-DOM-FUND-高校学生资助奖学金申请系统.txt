本科毕业设计（论文）开题报告

题目：基于 Spring Boot 与 Vue 的高校学生资助奖学金申请系统的设计与实现

开题年份：2025

一、选题背景与意义

1.1 选题背景
高校学生资助工作覆盖国家助学金、校级奖学金、困难补助等多个项目。每学年申报季，学工处与各学院辅导员需要收齐申请表、家庭经济困难认定材料及佐证复印件，再按学院汇总、会签后报送学校。纸质材料在传递中易遗失，学生对审核进度往往只能反复打电话询问，窗口压力大。

同时，不同资助项目的申报条件、材料清单与公示要求并不相同。若仍用表格与微信群收集，容易出现漏交材料、重复填报、公示名单难追溯等问题。建设面向学生与学工管理人员的资助申请信息系统，可将项目发布、在线填报、审核办结与结果查询集中到同一平台。

1.2 选题意义
（1）规范奖助学金与困难补助的申报、审核流程；
（2）提高材料完整性与审核留痕水平，减少反复补件；
（3）便于学生查询本人申请进度与结果；
（4）完成 Web 管理信息系统的需求分析、设计与实现训练。

二、国内外研究现状

国外高校学生事务系统常将资助、注册与成绩集成，权限与合规要求较高。国内学工一体化平台功能较全，但对本科毕设而言体量过大。本课题聚焦「资助项目—申请单—审核办结」主线，采用常见 Spring Boot 与 Vue 技术栈实现可演示的轻量系统。

三、研究目标与主要内容

3.1 研究目标
设计并实现基于 Spring Boot 与 Vue 的高校学生资助奖学金申请系统，支持学生与学工管理人员等角色，完成资助项目维护、在线申请、审核通过或驳回、申请记录查询以及公告发布。

3.2 主要功能
1. 用户注册登录与个人资料维护；
2. 资助/奖学金项目分类浏览与条件说明查看；
3. 学生在线填写并提交资助申请；
4. 管理人员审核通过或驳回，办结申请；
5. 本人申请记录与结果查询；
6. 校园公告的发布与查阅；
7. 管理端待审与办结情况概览。

3.3 拟解决关键问题
（1）申请单状态在待审、通过、驳回之间清晰流转；
（2）不同项目的材料说明与申请记录可关联查询；
（3）学生端与管理端菜单、权限分离。

四、技术方案

后端采用 Spring Boot、Spring Security、MyBatis 与 MySQL；前端采用 Vue 3、Element Plus，统计图表可使用 ECharts。系统采用 B/S 分层结构，接口与页面按角色授权访问。

五、非本期范围

与银行或财政直发系统对接、学工画像与大数据风控、电子签章与影像高拍仪硬件采集等不在本期实现；材料附件以系统内上传演示为主。

六、进度安排

第 1–2 周：调研资助申报流程、查阅文献并完成开题；
第 3–4 周：完成需求规格、用例与数据库设计；
第 5–10 周：实现项目维护、申请与审核等核心模块；
第 11–12 周：联调测试与演示数据准备；
第 13–16 周：撰写论文并准备答辩。

七、预期成果

可运行的 Web 系统、数据库脚本、毕业论文及答辩材料。
