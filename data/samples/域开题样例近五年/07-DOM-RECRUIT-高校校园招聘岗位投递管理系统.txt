本科毕业设计（论文）开题报告

题目：基于 Spring Boot 与 Vue 的高校校园招聘岗位投递管理系统的设计与实现

开题年份：2025

一、选题背景与意义

1.1 选题背景
每年春招秋招，就业指导中心要发布大量企事业单位岗位，学生通过邮箱、问卷或现场投递简历，材料格式不一，HR 初筛结果难以及时反馈到学生端。双选会当天排队投递效率低，部分岗位招满后仍有人重复投递。

院系就业干事需要掌握本专业投递与面试进度，但信息分散在各企业邮箱中。建设校园招聘岗位投递管理系统，可统一岗位发布、简历投递、初筛审核与投递记录查询。

1.2 选题意义
（1）提高校招信息触达与投递效率；
（2）企业/管理员可在线初筛并反馈状态；
（3）学生可跟踪投递记录，减少重复劳动；
（4）支撑就业信息化方向的毕设实践。

二、国内外研究现状

智联、前程无忧等面向社会招聘，校招场景需与学校认证、院系权限结合。学校就业网部分仅做信息发布不含审核流。本科系统聚焦岗位档案、简历投递与初筛审核的轻量实现。

三、研究目标与主要内容

3.1 研究目标
实现招聘岗位发布、学生简历投递、管理员初筛审核、投递记录查询及公告管理。

3.2 主要功能
1. 学生与企业管理员登录与资料维护；
2. 招聘岗位分类发布与浏览检索；
3. 学生选择岗位提交简历投递申请；
4. 管理员初筛审核（通过/不合适/待面试）；
5. 投递记录与状态查询；
6. 公告与就业政策发布。

3.3 拟解决关键问题
（1）同一岗位重复投递控制与状态展示；
（2）岗位关闭后停止新投递；
（3）按院系或专业筛选可见岗位（可选简化）。

四、技术方案

后端采用 Spring Boot 构建 REST 接口，Spring Security 实现登录鉴权与角色权限；持久层使用 MyBatis 访问 MySQL 数据库。前端采用 Vue 3 与 Element Plus 搭建管理端与业务页面，ECharts 用于工作台等业务统计图表。系统按表现层、业务层、数据访问层分层设计，便于模块开发与联调。

五、非本期范围

视频面试、ATS 简历解析、爬虫批量导入企业岗位不在本期。

六、进度安排

第 1–2 周：文献调研、需求梳理并完成开题答辩；
第 3–4 周：完成系统总体设计、数据库 ER 图与接口说明；
第 5–10 周：分模块编码实现核心业务与权限控制；
第 11–12 周：功能测试、缺陷修复与演示数据准备；
第 13–16 周：撰写毕业论文、整理答辩 PPT 与操作说明。

七、预期成果

（1）可本地部署运行的 Web 管理系统；
（2）MySQL 建表脚本与示例数据；
（3）符合学院规范的本科毕业论文一份；
（4）系统演示视频或答辩材料。
