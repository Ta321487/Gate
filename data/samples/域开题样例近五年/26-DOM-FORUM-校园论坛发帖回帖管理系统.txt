本科毕业设计（论文）开题报告

题目：基于 Spring Boot 与 Vue 的校园论坛发帖回帖管理系统的设计与实现

开题年份：2022

一、选题背景与意义

1.1 选题背景
我校学生交流仍依赖多个 QQ 群，信息碎片化且难以检索。曾有交易纠纷帖被删无据可查。学生会希望重建校园 BBS，按板块发帖，回帖需版主审核后展示，避免广告与不当言论。

论坛需支持主帖发布、回帖、一层楼中楼引用，管理员管理板块与审核。

1.2 选题意义
（1）集中校园讨论，便于检索沉淀；
（2）回帖审核提升内容质量；
（3）板块划分清晰，方便管理；
（4）训练社区类 UGC 系统设计。

二、国内外研究现状

Discuz 等传统论坛功能重，本科毕设常做简化 Web 版发帖回帖审核。

三、研究目标与主要内容

3.1 研究目标
实现论坛板块、主帖发布、回帖提交与审核、楼中楼一层引用及公告。

3.2 主要功能
1. 用户注册登录；
2. 论坛板块维护与浏览；
3. 主帖发布（标题、内容、板块）；
4. 回帖提交与版主审核；
5. 一层楼中楼引用回复；
6. 帖子搜索与个人发帖历史；
7. 公告与版规。

3.3 拟解决关键问题
（1）未审核回帖不对其他用户展示；
（2）楼中楼仅支持一层引用（防无限嵌套）；
（3）板块管理员权限隔离。

四、技术方案

后端采用 Spring Boot 构建 REST 接口，Spring Security 实现登录鉴权与角色权限；持久层使用 MyBatis 访问 MySQL 数据库。前端采用 Vue 3 与 Element Plus 搭建管理端与业务页面，ECharts 用于工作台等业务统计图表。系统按表现层、业务层、数据访问层分层设计，便于模块开发与联调。

五、非本期范围

实时私信、无限深度楼中楼树、富文本协同编辑、积分商城不在本期。

六、进度安排

第 1–2 周：文献调研、需求梳理并完成开题答辩；
第 3–4 周：完成系统总体设计、数据库 ER 图与接口说明；
第 5–10 周：分模块编码实现核心业务与权限控制；
第 11–12 周：功能测试、缺陷修复与演示数据准备；
第 13–16 周：撰写毕业论文、整理答辩 PPT 与操作说明。

七、预期成果

（1）可本地部署运行的 Web 管理系统；
（2）MySQL 建表脚本与示例数据；
（3）符合学院规范的本科毕业论文一份；
（4）系统演示视频或答辩材料。
