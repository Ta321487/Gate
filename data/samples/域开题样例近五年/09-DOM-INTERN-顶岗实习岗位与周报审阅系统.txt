本科毕业设计（论文）开题报告

题目：基于 Spring Boot 与 Vue 的顶岗实习岗位与周报审阅系统的设计与实现

开题年份：2024

一、选题背景与意义

1.1 选题背景
高职本科顶岗实习要求学生每周提交实习周报，校内指导教师与企业导师分别审阅。以往周报通过 QQ 或邮件提交，格式不统一，教师反馈分散，学院难以统计哪些学生逾期未交。

实习岗位信息、起止时间、企业联系人等档案也分散在表格中。需要一个系统集中维护实习岗位、接收周报并走审阅流程。

1.2 选题意义
（1）规范周报提交与导师审阅流程；
（2）实习岗位档案集中管理；
（3）逾期未交周报可提醒与统计；
（4）体现产教融合信息化管理实践。

二、国内外研究现状

部分院校使用实习管理平台，但轻量自建系统仍常见。本科毕设聚焦岗位档案、周报提交与审阅审核，不做三方协议电子签。

三、研究目标与主要内容

3.1 研究目标
实现实习岗位档案、实习周报提交、校内导师审阅审核、审阅意见反馈及公告管理。

3.2 主要功能
1. 学生、导师、管理员登录与资料；
2. 实习岗位档案维护（企业、岗位、时间）；
3. 学生按周提交实习周报；
4. 导师审阅并给出通过/修改意见；
5. 周报历史与审阅记录查询；
6. 公告与实习通知发布。

3.3 拟解决关键问题
（1）同一实习周期内周报按周次区分；
（2）审阅意见对学生可见并留痕；
（3）岗位与学生绑定关系清晰。

四、技术方案

后端采用 Spring Boot 构建 REST 接口，Spring Security 实现登录鉴权与角色权限；持久层使用 MyBatis 访问 MySQL 数据库。前端采用 Vue 3 与 Element Plus 搭建管理端与业务页面，ECharts 用于工作台等业务统计图表。系统按表现层、业务层、数据访问层分层设计，便于模块开发与联调。

五、非本期范围

三方协议电子签、GPS 签到、企业 ERP 对接不在本期。

六、进度安排

第 1–2 周：文献调研、需求梳理并完成开题答辩；
第 3–4 周：完成系统总体设计、数据库 ER 图与接口说明；
第 5–10 周：分模块编码实现核心业务与权限控制；
第 11–12 周：功能测试、缺陷修复与演示数据准备；
第 13–16 周：撰写毕业论文、整理答辩 PPT 与操作说明。

七、预期成果

（1）可本地部署运行的 Web 管理系统；
（2）MySQL 建表脚本与示例数据；
（3）符合学院规范的本科毕业论文一份；
（4）系统演示视频或答辩材料。
