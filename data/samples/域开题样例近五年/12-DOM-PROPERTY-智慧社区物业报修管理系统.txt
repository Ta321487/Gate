本科毕业设计（论文）开题报告

题目：基于 Spring Boot 与 Vue 的智慧社区物业报修管理系统的设计与实现

开题年份：2025

一、选题背景与意义

1.1 选题背景
本人实习所在社区共有十余栋住宅，业主报修渠道包括电话、业主群与前台登记，信息分散。物业维修班长手工排工，业主不知道师傅何时上门，重复投诉同一漏水问题因无工单编号无法关联。

社区希望业主可通过 Web 提交小区报修，物业前台审核派单，维修完成后线上确认。尤其电梯、公共照明、门禁故障需要留痕备查。

1.2 选题意义
（1）提升社区物业报修响应透明度；
（2）维修工单数字化，便于考核时效；
（3）业主可查询历史报修记录；
（4）训练社区 O2O 服务类系统设计。

二、国内外研究现状

大型物管公司使用收费工单系统，中小社区多采用微信接龙。本科毕设实现标准报修—派单—完结流程即可，不涉及多物业集团 ERP。

三、研究目标与主要内容

3.1 研究目标
实现业主报修提交、物业审核派单、维修处理完结及公告管理。

3.2 主要功能
1. 业主注册登录与房号资料；
2. 小区报修/维修工单提交（位置、问题、图片）；
3. 物业前台审核与派单；
4. 维修人员更新处理进度；
5. 完结确认与评价（简化）；
6. 公告与停水停电通知。

3.3 拟解决关键问题
（1）工单与房号、楼栋关联；
（2）派单后状态变更通知业主端展示；
（3）公共部位与私有部位报修分类。

四、技术方案

后端采用 Spring Boot 构建 REST 接口，Spring Security 实现登录鉴权与角色权限；持久层使用 MyBatis 访问 MySQL 数据库。前端采用 Vue 3 与 Element Plus 搭建管理端与业务页面，ECharts 用于工作台等业务统计图表。系统按表现层、业务层、数据访问层分层设计，便于模块开发与联调。

五、非本期范围

智慧门禁硬件、多物业公司集团 ERP、在线支付物业费不在本期。

六、进度安排

第 1–2 周：文献调研、需求梳理并完成开题答辩；
第 3–4 周：完成系统总体设计、数据库 ER 图与接口说明；
第 5–10 周：分模块编码实现核心业务与权限控制；
第 11–12 周：功能测试、缺陷修复与演示数据准备；
第 13–16 周：撰写毕业论文、整理答辩 PPT 与操作说明。

七、预期成果

（1）可本地部署运行的 Web 管理系统；
（2）MySQL 建表脚本与示例数据；
（3）符合学院规范的本科毕业论文一份；
（4）系统演示视频或答辩材料。
