本科毕业设计（论文）开题报告

题目：基于 Spring Boot 与 Vue 的校园失物招领管理系统的设计与实现

开题年份：2022

一、选题背景与意义

1.1 选题背景
校园内遗失物品与拾获物品的信息以往贴在食堂、图书馆布告栏，或发在校园墙 QQ 群，信息滚动快，失主难以检索，认领时也缺少身份核验记录。

保卫处与后勤希望统一线上失物招领平台：拾获者发布失物信息，失主提交认领申请并附证明，管理员审核后完成认领。

1.2 选题意义
（1）提高失物找回率，减少重复咨询；
（2）认领流程留痕，降低冒领风险；
（3）分类检索便于快速匹配；
（4）训练信息发布与审核类系统设计。

二、国内外研究现状

各地警方与高校有微信失物招领小程序，功能类似。本科 Web 系统实现发布、认领申请与审核即可。

三、研究目标与主要内容

3.1 研究目标
实现失物信息发布与分类、认领申请与审核、认领记录查询及公告管理。

3.2 主要功能
1. 用户登录与资料；
2. 失物/拾获信息发布与分类浏览；
3. 失主提交认领申请（可附证明材料）；
4. 管理员审核认领并办结；
5. 已认领/待认领状态展示；
6. 公告发布。

3.3 拟解决关键问题
（1）同一失物避免重复认领通过；
（2）认领审核要求简要身份说明；
（3）过期未认领信息可归档。

四、技术方案

后端采用 Spring Boot 构建 REST 接口，Spring Security 实现登录鉴权与角色权限；持久层使用 MyBatis 访问 MySQL 数据库。前端采用 Vue 3 与 Element Plus 搭建管理端与业务页面，ECharts 用于工作台等业务统计图表。系统按表现层、业务层、数据访问层分层设计，便于模块开发与联调。

五、非本期范围

图像识别自动匹配、与公安失物系统对接不在本期。

六、进度安排

第 1–2 周：文献调研、需求梳理并完成开题答辩；
第 3–4 周：完成系统总体设计、数据库 ER 图与接口说明；
第 5–10 周：分模块编码实现核心业务与权限控制；
第 11–12 周：功能测试、缺陷修复与演示数据准备；
第 13–16 周：撰写毕业论文、整理答辩 PPT 与操作说明。

七、预期成果

（1）可本地部署运行的 Web 管理系统；
（2）MySQL 建表脚本与示例数据；
（3）符合学院规范的本科毕业论文一份；
（4）系统演示视频或答辩材料。
