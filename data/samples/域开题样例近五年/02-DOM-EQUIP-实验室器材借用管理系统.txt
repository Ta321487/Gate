本科毕业设计（论文）开题报告

题目：基于 Spring Boot 与 Vue 的实验室器材借用管理系统的设计与实现

开题年份：2023

一、选题背景与意义

1.1 选题背景
电子信息学院多个专业共用基础实验室，示波器、信号源、开发板等器材价值较高且数量有限。以往借用靠实验室本登记，经常出现「登记了但未归还」「同一时段被多人预约」的情况，实验课前教师难以确认设备是否到位。

器材损坏与丢失后责任追溯困难，管理员月底汇总台账耗时且数据不准。亟需一套线上借用管理系统，让师生可查库存、走借用审核、按时归还并留痕。

1.2 选题意义
（1）提高实验室器材利用率，减少闲置与重复采购；
（2）借用与归还线上留痕，便于责任认定；
（3）支持借用额度与逾期提醒，规范教学秩序；
（4）锻炼实验室信息化管理系统的分析与实现能力。

二、国内外研究现状

国内外高校实验中心多配有 LIMS 或资产管理模块，但中小型院系往往无力采购全套系统。开源方案偏重库存而非「借用审核流」。本科毕设可聚焦器材档案、借用申请、审核归还等主路径，形成可演示的 Web 系统。

三、研究目标与主要内容

3.1 研究目标
实现实验室器材借用管理系统，覆盖器材分类档案、借用申请与审核、归还确认、可借数量占用、逾期提醒及公告管理。

3.2 主要功能
1. 用户登录与个人资料维护；
2. 实验室器材分类与设备档案检索；
3. 器材借用申请、管理员审核与借出登记；
4. 归还确认与器材状态更新；
5. 借用额度控制与逾期提醒；
6. 借用记录查询与器材使用统计；
7. 公告发布。

3.3 拟解决关键问题
（1）同一器材在同一时段避免重复借出；
（2）审核通过后占用可借数量并在归还后释放；
（3）逾期记录可供管理员催还与统计。

四、技术方案

后端采用 Spring Boot 构建 REST 接口，Spring Security 实现登录鉴权与角色权限；持久层使用 MyBatis 访问 MySQL 数据库。前端采用 Vue 3 与 Element Plus 搭建管理端与业务页面，ECharts 用于工作台等业务统计图表。系统按表现层、业务层、数据访问层分层设计，便于模块开发与联调。

五、非本期范围

物联网传感器盘点、RFID 全链路定位、设备远程控制等硬件对接不在本期。

六、进度安排

第 1–2 周：文献调研、需求梳理并完成开题答辩；
第 3–4 周：完成系统总体设计、数据库 ER 图与接口说明；
第 5–10 周：分模块编码实现核心业务与权限控制；
第 11–12 周：功能测试、缺陷修复与演示数据准备；
第 13–16 周：撰写毕业论文、整理答辩 PPT 与操作说明。

七、预期成果

（1）可本地部署运行的 Web 管理系统；
（2）MySQL 建表脚本与示例数据；
（3）符合学院规范的本科毕业论文一份；
（4）系统演示视频或答辩材料。
