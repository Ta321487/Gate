本科毕业设计（论文）开题报告

题目：基于 Spring Boot 与 Vue 的校园停车场车位预约系统的设计与实现

开题年份：2024

一、选题背景与意义

1.1 选题背景
我校东校区停车场车位有限，外来车辆与教职工临停争抢车位，保安手工登记入场，高峰无位可停引发纠纷。部分会议活动需预留车位，缺乏提前预约手段。

教职工与访客希望提前查看停车场剩余车位并预约时段，入场后核销释放。系统可统计各时段占用率，辅助管理。

1.2 选题意义
（1）提高车位利用率和预约透明度；
（2）减少门口拥堵与人工登记；
（3）预约占坑规则清晰可审计；
（4）体现预约资源管理开发实践。

二、国内外研究现状

商业智慧停车含车牌识别与道闸，成本高。本科 Web 系统做车位档案、时段预约与核销即可。

三、研究目标与主要内容

3.1 研究目标
实现停车场车位资源浏览、车位时段预约、取消与入场办结及公告管理。

3.2 主要功能
1. 用户登录与车辆资料（可选）；
2. 停车场与车位资源档案；
3. 选择时段提交车位预约；
4. 取消预约与占坑释放；
5. 入场核销与预约完结；
6. 预约记录与占用统计；
7. 公告发布。

3.3 拟解决关键问题
（1）同一车位同一时段不重复预约；
（2）预约超时未入场可自动释放（规则可配置）；
（3）车位状态实时展示。

四、技术方案

后端采用 Spring Boot 构建 REST 接口，Spring Security 实现登录鉴权与角色权限；持久层使用 MyBatis 访问 MySQL 数据库。前端采用 Vue 3 与 Element Plus 搭建管理端与业务页面，ECharts 用于工作台等业务统计图表。系统按表现层、业务层、数据访问层分层设计，便于模块开发与联调。

五、非本期范围

车牌识别闸机、无感支付、城市级停车平台对接不在本期。

六、进度安排

第 1–2 周：文献调研、需求梳理并完成开题答辩；
第 3–4 周：完成系统总体设计、数据库 ER 图与接口说明；
第 5–10 周：分模块编码实现核心业务与权限控制；
第 11–12 周：功能测试、缺陷修复与演示数据准备；
第 13–16 周：撰写毕业论文、整理答辩 PPT 与操作说明。

七、预期成果

（1）可本地部署运行的 Web 管理系统；
（2）MySQL 建表脚本与示例数据；
（3）符合学院规范的本科毕业论文一份；
（4）系统演示视频或答辩材料。
