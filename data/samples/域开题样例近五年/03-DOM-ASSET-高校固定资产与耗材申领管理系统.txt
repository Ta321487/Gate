本科毕业设计（论文）开题报告

题目：基于 Spring Boot 与 Vue 的高校固定资产与耗材申领管理系统的设计与实现

开题年份：2024

一、选题背景与意义

1.1 选题背景
学院行政与实验中心每年采购大量固定资产与实验耗材，以往台账分散在 Excel 与纸质领用单中。各教研室申领硒鼓、试剂、办公用品时，需反复找管理员核对库存，审批通过后出库登记再次手工誊写，容易出现超领、账实不符。

固定资产编号、存放地点、使用部门等信息更新滞后，年终盘点工作量大。建设统一的固定资产与耗材申领管理系统，有助于台账集中、申领在线审批、库存数量实时占用。

1.2 选题意义
（1）统一固定资产与耗材台账，降低账实不符风险；
（2）申领审批线上化，缩短物资发放周期；
（3）库存占用与回补可追踪，辅助采购计划；
（4）体现管理信息系统在高校后勤场景的应用。

二、国内外研究现状

大型单位多采用 ERP 或专业资产管理系统，功能涵盖采购、财务、多仓调拨，实施成本高。本科毕设适合实现「档案 + 申领审核 + 库存占用」的轻量 Web 版，不做完整进销存财务一体化。

三、研究目标与主要内容

3.1 研究目标
实现固定资产与耗材分类档案、物资申领申请与审核出库、库存数量占用与回补、申领记录查询及公告管理。

3.2 主要功能
1. 用户登录与部门资料维护；
2. 固定资产、耗材分类与物资档案维护；
3. 物资申领申请与管理员审核；
4. 审核通过后出库登记并占用库存；
5. 退库或回补更新库存数量；
6. 申领与库存变动记录查询；
7. 公告与用户管理。

3.3 拟解决关键问题
（1）申领审核与库存扣减在同一事务中保持一致；
（2）区分固定资产与耗材的不同登记字段；
（3）库存不足时给出明确提示并阻止超领。

四、技术方案

后端采用 Spring Boot 构建 REST 接口，Spring Security 实现登录鉴权与角色权限；持久层使用 MyBatis 访问 MySQL 数据库。前端采用 Vue 3 与 Element Plus 搭建管理端与业务页面，ECharts 用于工作台等业务统计图表。系统按表现层、业务层、数据访问层分层设计，便于模块开发与联调。

五、非本期范围

多仓 ERP、RFID 全链路盘点、财务凭证对接、采购招标模块不在本期。

六、进度安排

第 1–2 周：文献调研、需求梳理并完成开题答辩；
第 3–4 周：完成系统总体设计、数据库 ER 图与接口说明；
第 5–10 周：分模块编码实现核心业务与权限控制；
第 11–12 周：功能测试、缺陷修复与演示数据准备；
第 13–16 周：撰写毕业论文、整理答辩 PPT 与操作说明。

七、预期成果

（1）可本地部署运行的 Web 管理系统；
（2）MySQL 建表脚本与示例数据；
（3）符合学院规范的本科毕业论文一份；
（4）系统演示视频或答辩材料。
