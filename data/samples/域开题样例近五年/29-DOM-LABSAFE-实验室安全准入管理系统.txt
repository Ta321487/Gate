本科毕业设计（论文）开题报告

题目：基于 Spring Boot 与 Vue 的实验室安全准入管理系统的设计与实现

开题年份：2024

一、选题背景与意义

1.1 选题背景
工科院校实验室承担大量教学实验与创新训练。新进实验室的学生通常需先完成安全培训、熟悉操作规程，并经实验室负责人确认后，方可进入相应实验室开展实验。不少学院仍以纸质准入表、签字栏和口头通知为主，培训是否完成、准入是否到期、哪位同学被批准进入哪间实验室等信息分散，复查困难。

当实验课集中开出或开放实验室预约增多时，管理员难以及时掌握准入名单，存在未培训先入室、过期仍可进入等管理风险。建设实验室安全准入管理系统，可以把实验室档案、准入申请与审核办结放到线上，形成可查询的准入台账。

1.2 选题意义
（1）落实实验室安全培训与准入审核制度；
（2）减少纸质流转与口头确认带来的遗漏；
（3）便于实验室管理员与学生双方查询准入状态；
（4）锻炼权限控制与审核流类 Web 系统的设计实现能力。

二、国内外研究现状

国外高校实验室管理系统常与门禁、化学品库存联动，建设成本高。国内安全教育平台侧重在线考试与学时统计，与具体实验室准入名单衔接不足。本课题在本科毕设范围内实现「实验室档案 + 准入申请审核」的信息化，不扩展危化品全链路与硬件开锁。

三、研究目标与主要内容

3.1 研究目标
设计并实现基于 Spring Boot 与 Vue 的实验室安全准入管理系统，支持学生与实验室管理人员，完成实验室信息维护、准入申请提交、安全审核与办结、准入记录查询及公告管理。

3.2 主要功能
1. 用户登录与个人资料维护；
2. 实验室档案分类维护与查询；
3. 学生提交实验室准入申请并填写说明；
4. 管理员审核通过或驳回，办结准入；
5. 准入记录查询与统计概览；
6. 安全须知类公告的发布与查阅。

3.3 拟解决关键问题
（1）准入申请与实验室档案正确关联；
（2）审核状态清晰，避免重复无效申请；
（3）管理端与学生端职责分离。

四、技术方案

Spring Boot + Spring Security + MyBatis + MySQL；前端 Vue 3 + Element Plus。分层实现业务接口与页面，保证基本安全认证。

五、非本期范围

门禁闸机/电子锁硬件对接、危化品全生命周期台账、在线考试题库引擎与学时学分认定系统不在本期实现。

六、进度安排

第 1–2 周：调研实验室准入制度并开题；
第 3–4 周：完成需求分析与库表设计；
第 5–10 周：编码实现档案、申请与审核；
第 11–12 周：测试与演示准备；
第 13–16 周：论文与答辩。

七、预期成果

可演示系统、数据库脚本、毕业论文与答辩材料。
