一、选题背景与意义

随着城镇化推进与住宅小区规模扩大，物业服务中的报修与维修反馈已成为业主日常高频诉求。传统模式下，业主多通过电话、纸质登记或微信群沟通报修，容易出现信息遗漏、进度不透明、事后难以核对等问题，既影响服务体验，也不利于物业方掌握维修工作开展情况。

将报修登记、处理跟进与小区通知纳入统一的信息平台，有助于规范服务流程、提高沟通效率。本课题面向中小型物业服务场景，拟设计并实现一套基于 B/S 架构的物业报修管理系统，作为毕业设计的工程实践课题，兼顾业务合理性与技术可完成性。


二、国内外研究现状简述

国外设施管理与物业信息化起步较早，商业软件功能较为完整，但采购与本地化成本较高，且往往面向大型综合体。国内智慧社区、物业服务平台发展较快，报修类功能常见于综合物业系统或移动端入口；高校毕业设计中，采用 Java Web、Spring Boot 等技术实现报修、工单类系统的课题也较为多见，相关技术路线相对成熟。

结合毕业设计周期与实现条件，本课题聚焦 Web 端的报修业务闭环与基础管理功能，在需求范围上控制复杂度，保证分析、设计、实现与测试各环节能够完整展开。


三、研究目标

1. 分析小区物业报修业务，梳理业主、维修人员与物业管理人员的主要使用场景。
2. 完成系统总体设计、功能模块划分与数据库设计。
3. 基于 Spring Boot 与 Vue 完成系统开发，实现报修申请、处理与基础信息管理等功能。
4. 进行功能测试，整理源代码、数据库脚本与毕业设计说明书，完成答辩准备。


四、研究内容与拟实现功能

（一）用户与权限

系统区分普通业主与物业侧管理人员。业主可注册登录、维护个人资料、提交报修并查看处理进度与小区公告；物业侧人员负责报修处理、基础信息维护及用户与公告管理。具体权限在详细设计阶段按职责进一步划分。

（二）主要功能

1. 用户注册、登录与个人资料维护。
2. 业主在线提交报修（含位置、类型与故障描述），查询本人报修记录。
3. 物业侧对待处理报修进行审核处理，并跟进至办结。
4. 楼栋、房屋及报修类别等基础信息维护。
5. 业主用户信息管理。
6. 小区公告的发布与查阅。
7. 管理端业务概览（如待办数量等）。

（三）研究重点

围绕报修信息的结构化登记、处理过程状态变化，以及不同用户可见范围与操作权限展开设计与实现，使系统能够支撑从报修提交到处理完成的基本业务过程。


五、关键问题与解决思路

1. 需求边界：以报修主流程和必要基础数据为主，避免一次纳入过多周边业务导致无法按期完成。
2. 数据组织：合理设计用户、房屋位置、报修单与公告等实体及其联系，保证数据一致性与查询方便。
3. 权限控制：按角色限制管理功能与业主功能入口，防止越权操作。
4. 前后端协作：明确接口划分与页面流程，保证联调与测试可执行。


六、技术路线

1. 体系结构：B/S，前后端分离。
2. 后端：Java、Spring Boot、MySQL。
3. 前端：Vue、Element Plus 等。
4. 开发过程：需求分析 → 系统设计 → 编码实现 → 测试修改 → 撰写说明书。


七、进度安排（参考）

第 1–2 周：查阅资料，完成开题，细化需求与用例。
第 3–4 周：完成总体设计、数据库设计与界面草案。
第 5–8 周：实现登录注册、报修业务、基础数据与公告等模块。
第 9–10 周：系统联调与测试，补充说明材料。
第 11–12 周：完善毕业论文（设计说明书），准备答辩。
