一、选题背景与意义

高校实验室、实训中心与院系器材室常备投影仪、单片机套件、测量仪器、运动器材等可外借设备。师生借用若依赖纸质登记本或分散的表格、群消息，容易出现库存不清楚、审批不及时、归还节点难催促、历史记录难核查等问题，既影响教学实验安排，也不利于管理部门掌握设备使用率与损耗情况。

建设面向校园场景的设备借用信息系统，将设备目录、借用申请、审核发放与归还催还集中管理，有助于规范借用流程、提高物资周转效率。本课题拟设计并实现一套基于 B/S 架构的实验室设备借用管理系统，作为毕业设计工程实践课题，突出需求分析、系统设计与实现能力训练。


二、国内外研究现状简述

资产与物资管理领域中，固定资产台账、实验室信息管理系统（LIMS）等商业方案功能较全，但部署成本与业务范围往往超出本科毕业设计可承受规模。国内高校信息化建设中，部分单位已提供仪器预约或物资申领入口；毕业设计选题里，基于 Java Web、Spring Boot 实现图书借阅、设备/器材借用类系统的案例较为常见，技术路线成熟、可参考性强。

本课题结合学校实验室与器材室场景，将研究范围限定在 Web 端的设备检索、借用申请与审核归还闭环及必要的基础信息管理，保证在有限周期内完成分析、设计、编码、测试与说明书撰写。


三、研究目标

1. 分析实验室/器材室设备借用业务，梳理借用师生与设备管理人员的使用需求。
2. 完成系统总体设计、模块划分与数据库设计。
3. 采用 Spring Boot 与 Vue 实现可运行系统，覆盖设备目录、借用申请、审核、归还与基础管理等功能。
4. 完成功能测试，整理源码、数据库脚本与毕业设计说明书，准备答辩。


四、研究内容与拟实现功能

（一）用户与权限

系统面向借用师生与设备管理人员。师生可注册登录、维护个人资料、检索设备并提交借用申请，查看审核与归还进度及相关通知；管理侧负责借用审核、库存与分类维护、逾期催还，以及用户与公告管理。权限细节在详细设计中按岗位职责划分。

（二）主要功能

1. 用户注册、登录与个人资料维护。
2. 设备分类浏览与条件检索，查看设备详情与可借库存。
3. 在线提交借用申请，查询本人借用记录。
4. 管理侧对待审申请进行通过或驳回，记录审核说明；已借出设备支持归还登记。
5. 对逾期未还情况进行提醒或标记，辅助催还。
6. 设备与分类等基础信息维护（含库存数量）。
7. 用户信息管理。
8. 实验室/器材室公告的发布与查阅。
9. 管理端业务概览（如待审数量、在借数量等）。

（三）研究重点

围绕设备库存与借用单状态变化（申请、审核、借出、归还/逾期），以及普通用户与管理人员的功能边界展开，形成从申请到归还的基本业务过程。


五、关键问题与解决思路

1. 需求范围控制：以设备目录、借用申请—审核—归还主流程及必要基础数据为主，避免一次纳入采购、维保、RFID 盘点等过重子系统。
2. 库存一致性：审核通过时扣减可借数量，归还时回补，防止超借；异常情况在设计中约定处理规则。
3. 借期与逾期：明确应还日期计算与逾期提醒方式，保证演示与说明书中说明清楚。
4. 权限与数据安全：区分师生端与管理端功能，防止越权访问与越权完结他人单据。
5. 前后端协作：明确接口与页面流程，保证联调测试可执行。


六、技术路线

1. 体系结构：B/S，前后端分离。
2. 后端：Java、Spring Boot、MySQL。
3. 前端：Vue、Element Plus 等。
4. 开发过程：需求分析 → 系统设计 → 编码实现 → 测试修改 → 撰写说明书。


七、进度安排（参考）

第 1–2 周：查阅资料，完成开题，细化需求与用例。
第 3–4 周：完成总体设计、数据库设计与界面草案。
第 5–8 周：实现登录注册、设备目录、借用审核归还、基础数据与公告等模块。
第 9–10 周：系统联调与测试，补充说明材料。
第 11–12 周：完善毕业论文（设计说明书），准备答辩。


八、预期成果

2. 毕业设计说明书一份。
3. 系统演示环境及主要操作说明。
